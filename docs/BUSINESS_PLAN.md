화가가 팔레트에서 자신의 그림에 어울리는 색을 고르듯, 각자의 개성·취향·이상형이라는 고유한 색을 찾아 조화로운 관계를 만든다. 다음 세 가지 핵심 가치를 변하지 않는 축으로 한다:

3. **의미 있는 관계** (Create Masterpiece) — 주선자 인센티브(레벨별 15-40% 분배) · 매칭 후 10일 쿨타임으로 집중 유도 · 사회적 책임감 기반 관계 형성

## 2. 문제 인식 (Problem)

### 2.1 시장 현황

- **국내 미혼 인구 증가** — 통계청 인구 조사 기준 25-39세 미혼 비율 2023년 약 47% 도달. 결혼 평균 연령 남성 33.7세 / 여성 31.3세 (2023). 자연스러운 만남 기회 감소 ↔ 데이팅 서비스 수요 ↑.
- **데이팅 앱 사용 정상화** — 2018년 이후 데이팅 앱 사용 거부감 ↓. 20대 후반 ~ 30대 후반 인구의 데이팅 앱 사용 경험률 50%+ (오픈서베이 2024).
- **국내 모바일 데이팅 앱 시장** — 약 **2,400억원** (2024, 추정치). 글로벌 시장은 약 60억 달러 (Match Group, Bumble 등 포함). 한국 시장 성장률 연 12% 추세.

### 2.2 기존 데이팅 앱의 구조적 문제

| 문제 | 사용자 영향 | 시장 데이터 |
|---|---|---|
| **신뢰 부재** — 익명·낯선 사람과 무한 스와이프. 매칭 상대의 신원·평판 검증 수단 부재 | 사기·잠수·돌려막기·외도 위험. 만남 자체에 대한 피로감 | 데이팅 앱 사용자 60%가 "허위 프로필 경험" (2024 한국언론진흥재단) |
| **외모·스펙 중심** — 사진 한 장 + 키·직업 줄어드는 차별. 깊은 가치관·성향 매칭 부재 | 표면적 매칭 → 만남 후 빠른 이탈. 매칭 풀의 양적 경쟁이 질적 만남을 압도 | 매칭 앱 평균 매칭 후 7일 내 이탈률 70%+ |
| **무한 스와이프 피로** — 사용자에게 끝없는 선택 부담 → 결정 회피 (paradox of choice) | 가입 → 사용 → 만족 → 결제 의향 단계마다 큰 이탈 | 데이팅 앱 첫 7일 retention 평균 ~25% |
| **주선자 부재** — 친구 추천(소개팅) 의 안전성을 디지털로 옮길 도구 부재 | 결혼정보회사(고비용·중년 타깃)와 데이팅앱(저신뢰·20대) 사이의 공백 | 25-35세 직장인의 약 41% 가 "지인 소개" 를 가장 신뢰 (잡코리아 2023) |

### 2.3 사용자 페인포인트 (Interview / Survey 종합)

> "데이팅 앱에서 만난 사람은 어디까지 진짜인지 모르겠어요."
> "친구가 소개해주면 마음이 놓이는데, 그런 친구가 매번 있는 건 아니죠."
> "프로필이 다 비슷해 보여요. 키·직업·사진. 그 사람의 성격이나 가치관은 안 보여요."
> "결혼정보회사는 너무 부담스럽고, 데이팅 앱은 너무 가볍고."

### 2.4 문제 정의 (One-liner)

> **"25-39세 직장인이 '믿을 수 있는 지인 추천' 의 안전망 안에서, '나의 성향·가치관에 맞는 사람' 을 만나는 디지털 도구가 없다."**

## 3. 실현 가능성 (Solution)

### 3.1 솔루션 개요 — 3축 차별화

#### ① 지인 네트워크 기반 (신뢰)

- **1촌 / 2촌 / 3촌** 단계로 매칭 풀 확장. 1촌 = 직접 친구, 2촌 = 친구의 친구.
- **2단계 승인 검증** — 사용자 A → 1촌 주선자(친구) → 수신자 B. 두 사람의 친구가 직접 "이 매칭이 맞다"고 승인해야 진행.
- **사회적 책임감 효과** — 주선자가 평판을 걸기 때문에 부적절한 매칭·잠수·외도 위험 자연 감소.

#### ② AI 양방향 이상형 매칭 (품질)

- **AI 인터뷰 기반 프로필 생성** — OpenAI gpt-4o-mini 가 사용자 답변을 분석해 자동으로 500자 이상 자기소개 + 8가지 색깔(Orange·Blue·Red·Pink·Green·Purple·Yellow·Gray) 중 1개로 분류.
- **양방향 임베딩 매칭 (ADR 0047 §B)** — 자기소개 / 이상형을 **별도 벡터** 로 임베딩 (text-embedding-3-small, 1,536차원). 4축 가중 점수로 매칭:

| 축 | 가중치 | 의미 |
|---|---|---|
| 상호 이상형 적합도 | **50%** | cos(A.이상형, B.자기소개) × cos(B.이상형, A.자기소개) — **양쪽 모두 적합** 해야 점수↑ (짝사랑 페널티) |
| 자기소개 유사도 | 15% | cos(A.자기소개, B.자기소개) — 가치관·결의 결 |
| 색 궁합 | 20% | 보완색 0.9 / 유사색 0.75 / 동일·대비 0.5-0.6 (디자인 색채학 룰 백엔드 이식) |
| 활동 모멘텀 | 15% | 최근 7일 매칭 시도·로그인 tanh 정규화 — 활성 사용자 우선 |

- **자정 KST 배치 사전 캐싱** — Top 8 후보에 대해 LLM 종합 분석 미리 생성, 사용자 진입 시 즉시 반환.
- **LLM 안전바 6중** — stub 모드 / SHA-256 캐시 / timeout 5s/15s / retry / fallback / audit log. LLM 비용은 구독 매출의 0.56% 수준.

#### ③ 주선자 인센티브 분배 (생태계)

- **레벨 시스템** — 누적 매칭 성공 건수로 5단계 (Bronze 0건 / Silver 3건 / Gold 6건 / Platinum 11건 / Diamond 21건+).
- **소개 요청 분배** — 사용자가 매칭 시도 시 100 물감(10,000원) 차감 → 주선자 등급별 15~40% 분배 (Diamond 40 물감 / Bronze 15 물감).
- **팁 90/10 분배** — 매칭 성공 후 사용자가 자유롭게 10~500 물감 팁. 주선자 90% / 플랫폼 10%.
- **마일스톤 보너스** — 1·5·10·20·50·100·150건 누계 도달 시 추가 보상 (총 +1,925 물감).
- **인센티브 정렬 핵심** — 주선자가 좋은 매칭을 만들수록 본인 수익 증가 + 명예(리그·랭킹) 상승. "외부 송금 유도" 어뷰징 방지를 위해 약관 §6 + 90/10 분배 + 차단·신고 시스템(ADR 0046).

#### 3.3.2 패키지 구조 (DDD 4-layer)

```
kr.ai.palette/
├── domain/         # 순수 비즈니스 로직 (프레임워크 무관)
├── persistence/    # JPA Entity, Mapper, Repository 구현
├── application/    # 유스케이스, 트랜잭션 경계
├── presentation/   # REST 컨트롤러, DTO
└── infrastructure/ # 외부 연동, 인증, 시크릿, 설정
```

Import 방향 강제: `domain ← persistence`, `domain ← application`, `application ← presentation`. domain 에서 Spring framework import 금지. CI 에 자동 검증 룰.

#### 3.3.3 팔레트픽 매칭 파이프라인 (ADR 0047 §B)

```
[ Stage 1: Candidate Pool ]    지인망 2촌 + 60일 이력 제외 + 매칭 받기 ON + 동성 제외 → ≤ 50
         ↓
[ Stage 2: Deterministic Score ]  4축 weighted 0~100 (mutualIdeal × intro × color × momentum)
         ↓ Top 8
[ Stage 3: LLM Compatibility ]  gpt-4o-mini JSON: summary / strengths / watchOuts / firstQuestion
         ↓ 자정 KST 배치 사전 캐시 (CompatibilityAnalysisEntity, inputsHash 무효화)
[ Stage 4: 일일 1-3장 사용자 노출 ]  DailyRecommendation (variant 태깅)
```

비용 모델: **1구독자당 LLM 비용 ≈ 168원/월** (구독료 29,900원의 0.56%). 캐시 hit 시 호출 0회. 정상 호조 0.7원/건.

### 3.4 현재 개발 진척도

- **백엔드** — 도메인 Aggregate 8개 + 컨트롤러 40+ endpoint. 결제 / 매칭 / 주선 / 신뢰안전 / LLM / 어드민 모두 동작.
- **프론트** — 50+ 화면, 신규 가입 → 매칭 → 결제 → 출금까지 골든패스 통과.
- **테스트** — JUnit 5 + Kotest (DescribeSpec) — 도메인 핵심 로직 단위 테스트 + 안전바 (LLM scorer) 행위 검증 + 컨트롤러 통합 테스트.

| **앱스토어 심사 거부** (계정 삭제·권한·외부 결제) | 출시 지연 | ✅ **이미 구현** — 계정 삭제 (Apple/Google 의무 2024 강화 반영), 권한 사용 문자열 (Info.plist/AndroidManifest), IAP 통합 계획 (B-103) |
| **신뢰 부재로 시드 부족** — 친구 0명 신규 가입자 이탈 | 가입 → 활성 전환 ↓ | ✅ **부분 구현** — 데모 풀 + 친구 초대 마법사 + 가입 환영 보너스 60 물감. 정식 출시 시 옵트인 토글로 익명 풀 확장 |
| **경쟁 서비스 대응** | 가격·기능 따라잡기 | 핵심 차별화 = ① 지인망 + ② AI 양방향 + ③ 주선자 인센티브. 한 가지 기능 따라잡기는 가능하나 3축 동시 복제는 어려움. 시장 선점 + 사용자 락인 (지인망 = 네트워크 효과) |
