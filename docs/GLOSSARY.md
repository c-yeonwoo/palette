# GLOSSARY — 팔레트 도메인 용어집

> SoT. 새 용어 도입 / 기존 용어 변경 시 이 문서를 먼저 갱신하고 ADR 추가.
>
> 핵심 원칙: **"주선"은 주선자(matchmaker)의 활동·역할에만 쓰고, 사용자가 누군가와 이어달라고 하는 행위는 "소개 요청"으로 통일한다.** (ADR 0024)

---

## Layer 1 — 액터 (Actors)

| 용어 | 영문 | 의미 |
|------|------|------|
| **사용자** | User | 팔레트 회원 (REGULAR / MATCHMAKER_ONLY) |
| **요청자** | Requester | 소개 요청을 보낸 사용자 |
| **대상자** | Target / Receiver | 소개 요청의 매칭 상대 |
| **주선자** | Matchmaker | 두 사람을 이어주는 사람 (자신의 지인 네트워크 활용) |
| **지인** | Friend | 나와 양방향 친구 관계를 맺은 사람 |

---

## Layer 2 — 두 개의 독립된 관계망 (절대 혼용 금지)

팔레트에는 성격이 완전히 다른 두 관계망이 있다. **(A) 지인 관계**는 "누구를 아는가", **(B) 소개**는 "누구를 이어주는가"이다.

### A. 지인 관계 (Friendship)

| 용어 | 백엔드 entity | 의미 | 흐름 |
|------|---------------|------|------|
| **지인 요청** | `Friendship` (status=PENDING) | X가 Y와 **양방향 친구 관계**를 맺기 위해 보내는 요청 (친구 코드 / 검색 기반) | X → Y, Y 수락 시 서로의 지인 |

흐름:
1. X가 친구 코드 입력 또는 검색으로 Y 발견
2. X → Y 지인 요청 (`POST /api/v1/friends/request/{targetUserId}`)
3. Y의 알림 + `FriendConnectScreen` "받은 지인 요청" 영역에 표시
4. Y 수락 (`PUT /api/v1/friends/request/{requestId}/accept`) → 양방향 `Friendship` 성립
5. 이후 X·Y는 서로의 지인 목록에 노출되고, 서로를 주선자로 활용 가능 (대칭)

### B. 소개 (Introduction)

누군가를 다른 누군가와 이어주는 모든 행위. 시작 주체에 따라 둘로 나뉜다.

| 용어 | 백엔드 entity | 시작 주체 | 의미 |
|------|---------------|-----------|------|
| **소개 요청** | `MatchmakingRequest` | 요청자 (당사자) | 요청자 A가 주선자 C를 통해 대상자 B와 이어지길 요청 (A → C → B, 2단계 승인) |
| **소개 제안** | `Nudge` | 주선자 (제3자) | 주선자 C가 자신의 두 지인 A·B를 "한번 이어볼까요?"라고 먼저 제안 |

"소개 요청" 흐름:
1. 요청자 A가 피드에서 대상자 B 발견
2. A·B의 공통 주선자 C가 있어야 요청 가능
3. A → C에게 "B와 이어주세요" 소개 요청 (열람권/포인트 차감)
4. C 승인 → B에게 전달
5. B 승인 → 매칭 성사 → C에게 성사 커미션

> ⚠️ **(A) 지인 요청**과 **(B) 소개 요청**은 도메인이 완전히 분리됨. 지인 요청은 "친구 맺기", 소개 요청은 "이어주기". 혼용 금지.
>
> ⚠️ ADR 0015 초안의 "지인 등록 요청 = `MatchmakerApplication`"은 오류 — 해당 entity는 없으며 양방향 친구 관계는 `Friendship`이 관리 (ADR 0016에서 정정).

## Layer 3 — 내 설정 토글 (혼동 핵심 해소)

마이페이지 "공개 설정"의 두 토글은 역할이 다르다. 과거 "소개받기 / 주선받기"로 비슷해 혼동을 유발 → 아래로 정정 (ADR 0024).

| 토글 (UI) | 백엔드 필드 | 의미 | 과거 명칭 |
|-----------|-------------|------|-----------|
| **프로필 공개** | `visibility` / `hiddenAt` | 내 프로필이 지인 피드에 노출될지 | "소개받기" / "피드 노출" |
| **소개 받기** | `isAcceptingMatches` | 지인이 보내는 **소개 요청**을 받을지 | "주선받기" |

- **프로필 공개 = ON, 소개 받기 = OFF** → 둘러보긴 하지만 소개 요청은 안 받는 상태 ("소개만 받기" 거부)
- 둘 다 ON → 노출 + 소개 요청 수신 (기본)

## 과금·보상 용어

| 용어 | 영문 | 의미 |
|------|------|------|
| **열람권** | View Ticket | 프로필 상세를 열람하기 위한 선불 티켓. 피드 기본카드(메인사진+기본정보)는 무료, **상세 열람 시 1장 차감** (ADR 0024) |
| **팔레트 픽 패스** | Palette Pick Pass | 월 구독(₩9,900/월). "팔레트 픽"(색 궁합 추천) 무제한 + 궁합 리포트 제공. 미구독 시 매일 1장 미리보기 (궁합 점수 티저, 상세 잠금). (백엔드 식별자는 `ai-signal`·`AiPassSubscription` 유지) |
| **감사 포인트** | Thank-you Points | 주선자가 지인 요청을 수락할 때 받는 즉시 보상 (100~500P) |
| **성사 커미션** | Match Commission | 매칭 성사 시 주선자가 받는 비율 보상 (30~50%, 등급별) |

## 보조 용어

| 용어 | 영문 | 의미 |
|------|------|------|
| **팔레트 픽** | Palette Pick | 홈 "지인 \| 팔레트 픽" 탭 중 우측 — AI 색 궁합 기반 추천 카드 (`DailyRecommendationEntity`). 색 궁합 점수로 정렬. (과거 "오늘의 추천" — '오늘의/AI 추천'은 진부해 폐기, ADR 0028) |
| **팔레트 매칭** | AI Recommendation | 시스템이 산출하는 매칭 후보 (팔레트 픽의 백엔드 개념) |
| **나의 색 / 컬러 타입** | Color Type | 8색 성격 메타포. AI가 근거·성향·이상형 유추와 함께 발급 (`ColorType`) |
| **궁합** | Compatibility | 두 사람 색 타입 간 어울림 점수 (`colorCompatibility`) |
| **친구의 친구** | 2-degree | 나 → 지인 → 그 지인의 지인 (과거 "2촌"). 그 너머는 "한 다리 더 건너" (과거 "3촌") |

## "주선" 단어 사용 규칙

| 맥락 | 쓸 단어 |
|------|---------|
| 주선자의 역할·화면·활동 | **주선** (주선자, 주선 대시보드, 주선 활동, 주선 성사) |
| 사용자가 누군가와 이어달라 요청 | **소개 요청** (❌ 주선 요청) |
| 주선자가 먼저 두 지인을 이어보자 제안 | **소개 제안** (❌ 연결 제안) |

## 금지/주의 단어

| ❌ 쓰지 말 것 | ✅ 대신 사용 | 이유 |
|---------------|---------------|------|
| 주선 요청 (사용자 행위) | 소개 요청 | "주선"은 주선자 역할어로 한정 (ADR 0024) |
| 주선받기 (토글) | 소개 받기 | 소개 요청 수신 토글임을 명확화 (ADR 0024) |
| 소개받기 / 피드 노출 (토글) | 프로필 공개 | 노출 여부 토글임을 명확화 (ADR 0024) |
| 연결 제안 | 소개 제안 | "소개" 패밀리로 통일 (ADR 0024) |
| 친구 요청 / 지인 등록 요청 | 지인 요청 | 1개 어휘로 통일 (ADR 0024) |
| 주선 신청 | 지인 요청 / 소개 요청 (맥락별) | "신청"·"요청" 중복 (ADR 0015) |
| 2촌 / 3촌 | 친구의 친구 / 한 다리 더 건너 | 촌수 표현 노후 (ADR 0024) |
| 프렌드 / friend (UI) | 지인 | 한글 통일 (ADR 0008) |
| Palette (UI) | 팔레트 | 한글 통일 |

## 화면 ↔ 도메인 매핑

| 화면 | 앱 내 탭/타이틀 | 다루는 도메인 |
|------|-----------------|---------------|
| `MainFeedScreen` | 홈 · "소개" | 탭: 지인 네트워크 피드(좌, 기본) + 팔레트 픽(우, AI 색 궁합) (소개 요청 시작점) |
| `ProfileDetailScreen` | — | 프로필 상세 열람(열람권) + 소개 요청 작성 |
| `IntroductionHistoryScreen` | "인연" | 요청자 관점 — 내가 보낸/받은 소개 요청 (추후 매칭 완료 시 채팅) |
| `ConnectorDashboard` | "주선" | 주선자 관점 — 받은 소개 요청 + 지인 목록 + 매칭 이력 |
| `FriendConnectScreen` | — | 지인 관리 — 친구 코드/검색/지인 요청 보내기 + 받은 지인 요청 수락 |
| `MatchmakerRewardScreen` | — | 주선자 등급·포인트·출금·리워드 안내 |
| `MyPageScreen` | "나" | 자기 프로필 / 공개 설정(프로필 공개·소개 받기) / 나의 색 |
| `ColorDetailScreen` | — | 나의 색 근거·성향·이상형 유추 |
