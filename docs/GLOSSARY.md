# GLOSSARY — 팔레트 도메인 용어집

> SoT. 새 용어 도입 / 기존 용어 변경 시 이 문서를 먼저 갱신하고 ADR 추가.
>
> 핵심 원칙: **"주선"은 주선자(matchmaker)의 활동·역할에만 쓰고, 사용자가 누군가와 이어달라고 하는 행위는 "소개 요청"으로 통일한다.** (ADR 0024)

---

## Layer 1 — 액터 (Actors)

| 용어 | 영문 | 의미 |
|------|------|------|
| **사용자** | User | 팔레트 회원 (REGULAR / MATCHMAKER_ONLY) |
| **요청자** | Requester | 소개 요청을 보낸 사용자 |
| **대상자** | Target / Receiver | 소개 요청의 매칭 상대 |
| **주선자** | Matchmaker | 두 사람을 이어주는 사람 (자신의 지인 네트워크 활용) |
| **지인** | Friend | 나와 양방향 친구 관계를 맺은 사람 |

---

## Layer 2 — 두 개의 독립된 관계망 (절대 혼용 금지)

팔레트에는 성격이 완전히 다른 두 관계망이 있다. **(A) 지인 관계**는 "누구를 아는가", **(B) 소개**는 "누구를 이어주는가"이다.

### A. 지인 관계 (Friendship)

| 용어 | 백엔드 entity | 의미 | 흐름 |
|------|---------------|------|------|
| **지인 요청** | `Friendship` (status=PENDING) | X가 Y와 **양방향 친구 관계**를 맺기 위해 보내는 요청 (친구 코드 / 검색 기반) | X → Y, Y 수락 시 서로의 지인 |

흐름:
1. X가 친구 코드 입력 또는 검색으로 Y 발견
2. X → Y 지인 요청 (`POST /api/v1/friends/request/{targetUserId}`)
3. Y의 알림 + `FriendConnectScreen` "받은 지인 요청" 영역에 표시
4. Y 수락 (`PUT /api/v1/friends/request/{requestId}/accept`) → 양방향 `Friendship` 성립
5. 이후 X·Y는 서로의 지인 목록에 노출되고, 서로를 주선자로 활용 가능 (대칭)

### B. 소개 (Introduction)

누군가를 다른 누군가와 이어주는 모든 행위. 시작 주체에 따라 둘로 나뉜다.

| 용어 | 백엔드 entity | 시작 주체 | 의미 |
|------|---------------|-----------|------|
| **소개 요청** | `MatchmakingRequest` | 요청자 (당사자) | 요청자 A가 주선자 C를 통해 대상자 B와 이어지길 요청 (A → C → B, 2단계 승인) |
| **소개 제안** | `Nudge` | 주선자 (제3자) | 주선자 C가 자신의 두 지인 A·B를 "한번 이어볼까요?"라고 먼저 제안 |

"소개 요청" 흐름:
1. 요청자 A가 피드에서 대상자 B 발견
2. A·B의 공통 주선자 C가 있어야 요청 가능
3. A → C에게 "B와 이어주세요" 소개 요청 (잔액 30 물감 차감 — ADR 0042)
4. C 승인 → B에게 전달
5. B 승인 → 매칭 성사 → C(주선자) 등급·명예 상승 (금전 보상 없음 — 무현금, ADR 0064)

> ⚠️ **(A) 지인 요청**과 **(B) 소개 요청**은 도메인이 완전히 분리됨. 지인 요청은 "친구 맺기", 소개 요청은 "이어주기". 혼용 금지.
>
> ⚠️ ADR 0015 초안의 "지인 등록 요청 = `MatchmakerApplication`"은 오류 — 해당 entity는 없으며 양방향 친구 관계는 `Friendship`이 관리 (ADR 0016에서 정정).

## Layer 3 — 내 설정 토글 (혼동 핵심 해소)

마이페이지 "공개 설정"의 두 토글은 역할이 다르다. 과거 "소개받기 / 주선받기"로 비슷해 혼동을 유발 → 아래로 정정 (ADR 0024).

| 토글 (UI) | 백엔드 필드 | 의미 | 과거 명칭 |
|-----------|-------------|------|-----------|
| **프로필 공개** | `visibility` / `hiddenAt` | 내 프로필이 지인 피드에 노출될지 | "소개받기" / "피드 노출" |
| **소개 요청 받기** | `isAcceptingMatches` | 지인이 보내는 소개 요청을 받을지 (수신자 토글) | "주선받기"(ADR 0024)/"주선 요청 받기" |

- **프로필 공개 = ON, 소개 요청 받기 = OFF** → 둘러보긴 하지만 요청은 안 받는 상태
- 둘 다 ON → 노출 + 소개 요청 수신 (기본)

## 과금·보상 용어

> **단일 잔액 모델 (ADR 0042)** — 모든 결제/보상은 **"물감"** 잔액 하나로 통합. 티켓 개념 폐기.
> 코드 변수는 `P` / `points` (Paint 의미) 유지. 사용자 노출은 "물감".

| 용어 | 영문 | 의미 |
|------|------|------|
| **물감** | Paint (P) | 단일 잔액 단위. `1 물감 = 100원`. 모든 사용처(열람·소개·팁) 자동 차감. 별풍선·카카오 초코와 같은 친숙한 단위감 + 브랜드(팔레트) 정체성. (ADR 0042) |
| **유료 물감** | paidPoints | 충전·정상 보상으로 적립. 만료 없음. 환불·출금 가능. (ADR 0042) |
| **보너스 물감** | bonusPoints | 가입·친구 가입·마일스톤 등 무상 지급분. `bonusExpiresAt` 만료. 소비 우선순위 (bonus → paid). 출금 불가. (ADR 0041 + 0042) |
| **친구의 친구 열람** | View FoF | 친구의 친구 프로필 상세 열람 — 잔액 **20 물감** (커피 한잔값, 2,000원) 자동 차감. 1촌은 무료. (ADR 0044) |
| **한 다리 더 건너 열람** | View 3-hop | **30 물감** (3,000원). 셀프 풀 확장(지인 추가) 인센티브로 의도적 차등. (ADR 0044) |
| **소개 요청** | Intro Request | 주선자에게 매칭 요청 시 잔액 **100 물감** (10,000원) 자동 차감. 진심 신호. 전액 플랫폼 수익 — 주선자 분배 없음 (무현금, ADR 0064). |
| ~~**성의 표시 (팁)**~~ | ~~Tip~~ | **제거됨** — 무현금 모델로 주선자 팁/분배 폐지 (ADR 0064). |
| **충전 묶음** | Point Bundle | 50 / 110 / 340 / 575 물감 4종 (5천~5만원). 큰 묶음일수록 보너스 % 증가. 110 묶음은 신규 트라이얼 반값 대상 (ADR 0045). |
| **팔레트 Pick 패스** | Palette Pick Pass | 월 구독(₩29,900/월). 색 궁합 AI 추천 무제한. **계정당 첫 달 무료 트라이얼** (ADR 0045). 별도 도메인(`AiPassSubscription`). |
| ~~**성사 분배**~~ | ~~Intro Commission~~ | **제거됨** — 주선자 금전 분배 폐지. 매칭 성사는 주선자 등급·명예에만 반영 (무현금, ADR 0064). |
| **가입 환영 보너스** | Welcome Bonus | 가입 직후 자동 지급 **60 물감** (친친 3명 열람 가치, 7일 만료). (ADR 0044) |
| **친구 가입 보너스** | Friend-Signup Bonus | 친구 수락 시 양쪽 모두 **10 물감** 지급 (14일 만료). (ADR 0041 + 0042) |
| **주선자 마일스톤** | Matchmaker Milestone | 누계 매칭 1·5·10·20·50·100·150건 도달 시 명예 배지·등급 반영 (금전·물감 보상 없음 — 무현금, ADR 0064). |

## 보조 용어

| 용어 | 영문 | 의미 |
|------|------|------|
| **팔레트 Pick** | Palette Pick | 홈 "지인 \| 팔레트 Pick" 탭 중 우측 — AI 색 궁합 기반 추천 카드 (`DailyRecommendationEntity`). 색 궁합 점수로 정렬. (과거 "오늘의 추천" — '오늘의/AI 추천'은 진부해 폐기, ADR 0028) |

## "주선" 단어 사용 규칙

| 맥락 | 쓸 단어 |
|------|---------|
| 주선자의 역할·화면·활동 | **주선** (주선자, 주선 대시보드, 주선 활동, 주선 성사) |
| 사용자가 누군가와 이어달라 요청 | **소개 요청** (❌ 주선 요청) |
| 주선자가 먼저 두 지인을 이어보자 제안 | **소개 제안** (❌ 연결 제안) |

| 화면 | 앱 내 탭/타이틀 | 다루는 도메인 |
|------|-----------------|---------------|
| `MainFeedScreen` | 홈 · "소개" | 탭: 지인 네트워크 피드(좌, 기본) + 팔레트 Pick(우, AI 색 궁합) (소개 요청 시작점) |
| `ProfileDetailScreen` | — | 프로필 상세 열람(친친 10 물감 차감, 1촌 무료) + 소개 요청 작성 |

## 변경 이력

- 2026-05-31 — 초안 작성. "주선 신청" → "지인 등록 요청" UI 어휘 일괄 변경 (ADR 0015).
- 2026-05-31 — entity 매핑 정정. `MatchmakerApplication` 부재, 양방향 친구 관계는 `Friendship`. ConnectorDashboard application UI 제거, `FriendConnectScreen` 일원화 (ADR 0016).
- 2026-06-07 — 사용자 행위 용어를 **"소개"로 최종 통일** (ADR 0032). 발신=소개 요청하기, 수신 토글=소개 요청 받기. **주선자 역할어(주선자/주선 대시보드/주선 활동)는 유지** — 주선=역할·서비스 층, 소개=사용자 경험 층으로 분리.
- 2026-06-10 — **단일 잔액 모델 "물감" 도입** (ADR 0042). 4축 결제(열람 티켓·소개 요청 티켓·팁·구독)를 잔액 하나(`물감`, 1 물감 = 100원)로 통합. 친친 열람 10 / 한 다리 더 건너 20 / 소개 요청 30 / 팁 10~100. ADR 0039 폐기, `TicketKind`·`TicketBundleCatalog` deprecated.
- 2026-06-10 — **가격 v2** (ADR 0044). 친친 열람 10→20, 한 다리 더 건너 20→30, 소개 요청 30→100, 팁 10~100→10~500. 소개 요청 분배 모델 (주선자 Lv별 15~40% / 플랫폼 60~85%). 팁 90/10 분배 도입. 환영 보너스 30→60 물감. 팔레트픽 9,900→29,900원 + 첫달 무료. **ADR 0045 신규 트라이얼 (열람 3일·반값 묶음·무료 소개 1회·팔레트픽 첫달)**, **ADR 0046 외부 송금 어뷰징 가드 (5단)** 신설.
- 2026-06-06 — 용어 전면 정리 (ADR 0024). "주선"을 주선자 역할어로 한정. 사용자 행위 "주선 요청"→"소개 요청", "연결 제안"→"소개 제안", "친구 요청/지인 등록 요청"→"지인 요청". 토글 "소개받기/주선받기"→"프로필 공개/소개 받기". 촌수 표현 "2촌/3촌"→"친구의 친구/한 다리 더 건너". 프로필 열람 티켓·AI 추천 구독 패스 용어 추가.
