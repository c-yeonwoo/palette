## P0 — 정식 출시 (3개월) 전 필수

### 주선자 정산 컴플라이언스 (CM) — 출시 블로커 (ADR 0064)

> 개인 주선자에게 현금을 지급하는 정산은 세무(원천징수)·전자금융거래법·결혼중개업/다단계 3개 레이어를 건드린다. 베타는 무현금(모델 B)로 운영하고, 정식 전환 전 자문으로 모델 확정. 상세·자문 질의서: `docs/DECISIONS/0064-matchmaker-settlement-compliance.md`.

#### CM-001. 주선자 보상 모델 결정 ⛔ (A 현금 / B 폐쇄형 무현금 / C P2P 자발 / B+C)
- ADR 0064 §6 체크리스트 D1~D7 확정. 변호사·세무사·금융규제 자문 선행.

#### CM-002. 다단계(방문판매법) 리스크 제거
- 레벨·친구초대 보상에서 *하위 가입 유도형 보상* 제거 → 직접 주선 성과 기반으로 단순화. 자문으로 후원방문판매 해당 여부 확인.

#### CM-003. 베타 = 무현금(폐쇄형) 전환
- 출금(`WithdrawalRequest`/출금 흐름)을 **기능 플래그로 비활성**(도메인 보존). 주선 보상을 *본인 데이팅 혜택(물감·열람권·구독 크레딧) + 명예(레벨·뱃지·성사 스토리)*로 재구성.
- 영향 문서: POLICY §2.6(출금)·§6(외부송금)·§7(중개업), PRD §7, ADR 0046.

#### CM-004. ADR 0046 / §6 외부송금 정책 재정의
- 무현금 모델 채택 시 90/10 인앱 분배 전제가 사라짐 → 외부송금 금지(§6) 유지/개정/폐기 결정. P2P QR 채택 시 "전적 자발·앱 비수취" UX·문구 필수.

#### CM-005. (모델 A 채택 시) 정산·세무 인프라
- 등록 PG/정산대행 + 긱워커(프리랜서 3.3%) 정산대행 선정, 원천징수·지급명세서 파이프라인, 물감 선불수단 등록 검토.

#### CM-006. 초기 주선자 공급 전략 (현금 유인 제거 시)
- 무현금 전환 시 부업형 헤비 주선자 동기 약화 → 호의/명예/본인혜택 기반으로 초기 주선자 풀 확보 방안 설계.

### 수익화

#### B-001. 신규 가입 7일 무료 체험 ⭐
- 가입 직후 자동 지급: 프로필 열람 티켓 3장 + 소개 요청 티켓 1장
- 만료 7일 (그 이후 잔여분 자동 소멸)
- 목표: 결제 진입 전 가치 학습 (Aha! moment)
- KPI: 가입 후 7일 내 매칭 시도율 / 결제 전환율

#### B-002. 친구 가입 보너스 ⭐
- 친구 1명 가입 시 양쪽에 열람권 1장 + 신뢰도 +5
- 어뷰징 방지: 동일 IP/디바이스에서 다수 가입 제한, 친구 수락 24h 내 신규 가입자만 카운트
- KPI: 가입자당 평균 친구 수, 1주차 retention

#### B-003. 주선자 마일스톤 보너스 ⭐
- 첫 매칭 +500P / 5건 누계 +2,000P / 10건 +5,000P / 20건 +10,000P / 50건 +25,000P
- 신규 친구 가입 시 +200P (B-002와 별개)
- 목표: Lv.1~2 주선자 동기 부여 (현재 매칭당 225~300P는 박함)
- KPI: 주선자 활성도 (월 매칭 시도 수), 다이아 도달율

#### B-004. 묶음 할인 결제 화면 노출
- 정책 §1.2b 의 1·5·10장 옵션을 결제 모달에 표시
- 5장 옵션에 "best value" 배지
- 베타엔 노출만 — 정식 결제 활성화 시 클릭 → IAP/Toss

### UX · Discover

#### B-005. 친구 초대 마법사 ⭐
- 가입 직후 5단계 온보딩 후 옵셔널 단계 추가: "지인 5명 초대로 시작"
- 카카오 친구 동기화 옵션 (옵셔널, 미동의 시 친구 코드만 노출)
- 스킵 가능, 단 "친구 가입 시 보상 받기" 안내

#### B-006. 데모 풀 (친구 0명일 때) ⭐ — 부분 구현 → P1
- ✅ 베타: `NoFriendsNudge` + 시드 유저 12명 (현재 동작)
- ⏸ 정식 출시: 옵트인 토글 ("신규 유저에게 내 프로필 익명 노출 OK")로 실 가입자 풀로 전환 → **B-114 (P1)**

#### B-007. 푸시 알림 트리거 5종 ⭐ — 일부 P0, 나머지 P1
- ✅ 친구 신규 가입 (`FRIEND_NEW_SIGNUP`) — 친구 수락 hook 즉시
- ✅ 매칭 후 D+1·D+3·D+7 (`POST_MATCH_NUDGE`) — MatchFollowUpScheduler 확장
- ⏸ 친구의 친구 풀 갱신 — 감지 비용 + 스팸 위험 평가 후 → **B-115**
- ⏸ 팔레트 Pick 일일 갱신 — 옵트인 정책 결정 후 → **B-116**
- ⏸ 주선 활동 주간 보고 — 집계 형식 결정 후 → **B-117**

### 채팅 (CH) — 핸드오프 안전 레이어 (ADR 0066)

> 성사 후 외부 채널(카카오톡/문자) 이탈 시 신고·차단·증거 보존이 닿지 않고, 즉시 전화번호 노출 마찰. 인앱 1:1 채팅으로 번호 없이 첫 대화 → 신뢰 후 ADR 0065 연락처 카드로 전환. **폴링 권고(WebSocket 아님)**. 착수 트리거: 베타에서 번호 교환 거부율·외부 분쟁 신고 관측 시 우선순위 상향.

- **CH-001. 데이터 모델·마이그레이션** — `chat_messages`(request_id·sender_id·body·created_at·read_at) + idempotent `CREATE TABLE` + `SchemaDriftTest` 등록.
- **CH-002. REST 폴링 엔드포인트** — `GET/POST /relationships/{id}/messages`. 당사자 403 가드(ADR 0065 패턴) + `COMPLETED` 체크 + `after={id}` 증분 조회.
- **CH-003. ChatThread 실배선** — mock(`VIEWER_ID="me-001"`) 제거, 폴링 연동. "인연" `MATCHED+` 카드에 채팅 진입 + 안 읽음 배지. ADR 0065 보존 `MatchDetailScreen`/`chat/*` 재활용/정리.
- **CH-004. moderation 연동** — 기존 `UserBlock`(양방향 스레드 숨김+전송 차단)·`UserReport`(스레드 신고→운영 큐) 재사용. `ENDED` 관계 읽기 전용 동결.
- **CH-005. 보존·삭제 잡** — `ENDED`/차단 후 90일 purge, 계정 삭제 시 즉시 purge(상대 스레드 발신분 포함).
- **CH-006. (후속) PII·외부송금 경고** — 발신 시 계좌번호/송금 유도 패턴 경고 배너 (ADR 0046 §6 연계).

### 내친소 게시판 (NC) — 지인 추천형 디스커버리 (ADR 0071) · 유동성 확보 후 착수

> "셀소" 트렌드를 **신뢰 기반(제3자 보증)**으로 변주. 주선자가 지인을 추천 게시 → 신청 → 선택 → 기존 2단계 매칭.
> 가드: 대상 옵트인 필수 · 사진 단계 공개 · 신청 물감 · 차단 격리 · 신고/모더레이션. (오픈 자기소개 게시판은 채택 안 함)

**Phase A — 백엔드 코어**
- **NC-A1. 도메인+영속화** — `IntroductionPost`/`IntroductionApplication`(+status enum, postType) + repo/entity/mapper + 마이그레이션 §28
- **NC-A2. 동의 플로우** — 주선자 게시(draft) → 대상 동의 알림(NotificationType 추가) → ACTIVE / 철회 시 비활성
- **NC-A3. 게시판 조회 API** — 활성 게시글 목록(필터·페이지·차단격리·본인/신청여부 표시) + 상세
- **NC-A4. 신청 API** — 물감 차감(BillingService) + rate-limit + 중복방지 + 신청 취소
- **NC-A5. 주선자 선택 API** — 신청자 1명 선택 → `MatchmakingRequest` 생성(기존 엔진 위임) + 나머지 정중 마감 알림

**Phase B — 프론트**
- **NC-B1. 게시판 화면** — 카드 리스트+필터(무한스크롤/좋아요 X — "구경" 방지)
- **NC-B2. 게시 생성** — 주선자: 지인 선택 → 추천사 작성 → 동의 요청
- **NC-B3. 동의 화면** — 대상: "내친소 등록 요청" 승인/거절
- **NC-B4. 신청·신청자 관리** — 주선자: 신청자 프로필 검토 → 선택
- **NC-B5. 진입점+알림** — 주선자 대시보드 + Discovery, 동의/선택/마감 알림 연동

**Phase C — 안전·정책·계측**
- **NC-C1. 신고 카테고리(동의없는게시)** + 어드민 모더레이션 큐
- **NC-C2. POLICY/약관 반영** (물감·rate-limit·동의)
- **NC-C3. 퍼널 지표** — 게시→신청→선택→성사 (어드민)

### WANTED (WT) — 이상형 역경매 (ADR 0071, Phase 3 실험)
- **WT-001.** `postType=WANTED` 분기 — 게시자=당사자(주선자·대상 없음), NC 의 신청·선택·프로필교환 머신 재사용
- **WT-002.** 별도 게시 흐름(원하는 결 작성) + 지원자 선택 → 게시자 프로필 전달 → 지원자 최종 결정
- **WT-003.** a/b — 유동성·성비·게시자 권력 집중 관측 후 정식화 판단
