# Palette — 운영 정책

> 결제·환불·광고·책임·개인정보 등 사용자에게 노출되는 운영 정책 명문화.
> 베타 단계와 정식 출시 단계를 분리해서 기록 — 베타 운영 중에 추가 검토/조정이 필요한 항목 명확히.
> **사람만 수정**. 정책 변경은 ADR 동반.

---

## 1. 결제 / 수수료

### 1.1 단일 잔액 모델 (ADR 0042 + 0044)

> **모든 결제는 단일 잔액(물감)에서 차감.** 1 물감 = 100원.
> 별풍선·카카오톡 초코 같은 친숙한 단위감 + 사용자 멘탈 모델 단순화.
> (이전 4축 티켓·포인트 분리 모델 = ADR 0039 폐기)

#### A. 사용처 — 잔액 자동 차감 (ADR 0044)

| 항목 | 차감 (물감) | 원화 환산 | 베타 | 정식 |
|---|---|---|---|---|
| 지인(1촌) 프로필 상세 열람 | **무료** | 무료 | ✅ | ✅ |
| 친구의 친구 프로필 상세 열람 | **20 물감** | 2,000원 (커피 한잔값) | ⚠️ 결제 미연동 — 모두 무료 | ✅ |
| 한 다리 더 건너 프로필 상세 열람 | **30 물감** | 3,000원 (셀프 풀 확장 인센티브) | ❌ Phase 2 | ✅ |
| **소개 요청 1건** | **100 물감** | 10,000원 (진심 신호) | ⚠️ 결제 미연동 | ✅ |

> 한 다리 더 건너가 더 비싼 이유: "돈 내기 싫으면 지인 더 추가하면 됨, 귀찮으면 돈 내고 풀 넓게 쓰셈" 의도적 가격 차등으로 셀프 풀 확장 유도.

#### B. 성의 표시 (옵셔널 tip) — 90/10 분배 (ADR 0044)

매칭 시도·성사 후 사용자가 **자유롭게** 주선자에게 보내는 감사 표시. 잔액에서 차감, **주선자 90% / 플랫폼 10%** 분배.

| 항목 | 범위 | 분배 | 베타 | 정식 |
|---|---|---|---|---|
| 주선자에게 성의 표시 | **10~500 물감** (1,000~50,000원) | 주선자 90 / 플랫폼 10 | ❌ Phase 2 | ✅ |

> 플랫폼 수수료 10% 는 외부 송금 어뷰징(ADR 0046) 방어 인센티브 정렬에 필수.
> 강제 아님 — 안 보내도 매칭은 정상 진행. 주선자 등급 통계에 미반영. 90% 만 출금 가능 잔액 적립.

#### C. 팔레트 Pick 패스 (월 구독, ADR 0044)

| 항목 | 가격 | 베타 | 정식 |
|---|---|---|---|
| **팔레트 Pick 패스** | **29,900원/월** | ❌ Phase 2 (개념 확정) | ✅ |
| **첫 달 무료 트라이얼** | 계정당 1회, 카드 등록 X, 30일 자동 만료 | ADR 0045 | ✅ |
| 팔레트 Pick 카드 (미구독) | 매일 1장 무료 미리보기, 2번째부터 잠금 | ⚠️ paymentKey 존재만 체크 (Toss 미검증) | ✅ |

**팔레트 Pick 패스(29,900원/월)**: 홈 "팔레트 Pick" 탭(색 궁합 기반) 무제한 + 궁합 리포트. 시장 평균(틴더 골드·글램·위피 15~20k) 상위 가격대 — AI 추천 + 데이터 기반 차별화 가치. 별도 도메인 — 잔액과 분리 (`AiPassSubscription`).

### 1.2 주선자 보상 — 소개요청 분배 모델 (ADR 0044)

ADR 0038 의 stand-alone reward (기준 20 물감 × 등급%) 폐기 → **소개 요청 100 물감 발생 시점에 등급별 분배**.

| 레벨 | 등급명 | 성공 매칭 누계 | 분배율 | 매칭당 보상 (소개요청 100 × %) |
|---|---|---|---|---|
| Lv.1 | 브론즈 | 0~2 | 15% | **15 물감** (1,500원) |
| Lv.2 | 실버 | 3~5 | 20% | **20 물감** (2,000원) |
| Lv.3 | 골드 | 6~10 | 25% | **25 물감** (2,500원) |
| Lv.4 | 플래티넘 | 11~20 | 30% | **30 물감** (3,000원) |
| Lv.5 | 다이아몬드 | 21+ | **40%** | **40 물감** (4,000원) |

> 단가 5배 인상 (30→100) + 분배 모델로 변경 → 다이아 매칭당 보상 8→40 물감 (**+400%**) + 플랫폼 마진 22→60 물감 (**+173%**) 동시 달성.
> 승급 기준은 ADR 0038 의 0/3/6/11/21 유지 (베타 운영 데이터 누적 후 재조정).
> 보상은 "현재 등급 기준" 적립 후 등급 갱신 — hot-streak 보호.

> 등급명·구간·커미션·색의 단일 소스: `frontend/src/lib/matchmakerLevels.ts`
> 백엔드 보상값 SoT: `domain/billing/Ticket.kt` (`PointPrice.INTRO_REQUEST`) + `domain/matchmaker/MatchmakerLevel.kt`

> 등급명·구간·커미션·색의 단일 소스: `frontend/src/lib/matchmakerLevels.ts`
> 백엔드 보상값 SoT: `domain/billing/Ticket.kt` + `MatchmakerMilestonePolicy`

- **출금 단위**: 1 물감 = 100원, 최소 출금 **50 물감** (= 5,000원)
- **출금 가능**: `paidPoints` (정상 보상 + 팁 수령). `bonusPoints` (보너스성)는 출금 X
- 출금 신청 → 본인 명의 계좌 (일·월 한도는 RUNBOOK 참조)

### 1.2b 충전 묶음 — ADR 0042

낱장 개념 폐기. 잔액(물감) 단위 충전 + 보너스 인센티브.

| 충전 묶음 | 결제 금액 | 보너스 | 인식 |
|---|---|---|---|
| 50 물감 | 5,000원 | — | 정가 |
| **110 물감** | 10,000원 | +10 물감 (10%↑) | "더 받음" |
| **340 물감** | 30,000원 | +40 물감 (13%↑) | BEST VALUE |
| **575 물감** | 50,000원 | +75 물감 (15%↑) | MAX 보너스 |

- 충전 잔액은 유효기간 없음 (만료 X).
- 매칭 성사 환불 시 차감된 물감 그대로 `paidPoints` 로 복귀.

### 1.2c 보너스 잔액 — ADR 0041/0042/0044

| 종류 | 지급 (물감) | 유효 | 트리거 |
|---|---|---|---|
| 가입 환영 보너스 | **60 물감** (친친 3명 열람 가치) | 7일 | 가입 직후 자동 |
| 친구 가입 보너스 (양쪽) | **10 물감** | 14일 | 친구 수락 시 |
| 주선자 마일스톤 1건 | +5 물감 | — | 첫 매칭 |
| 주선자 마일스톤 5건 | +20 물감 | — | 5건 누계 |
| 주선자 마일스톤 10/20/50/100/150건 | +50/100/250/500/1,000 물감 | — | 누계 도달 |

> 보너스는 보너스 잔액. 소비 순서: bonus → paid (사용자에 유리).

### 1.2d 신규 가입 트라이얼 — ADR 0045

가입 직후 자동 부여 (어뷰징 가드: 폰 인증 + 디바이스 핑거프린트 + 본인인증):

| 트라이얼 | 혜택 | 기간 | 어뷰징 방어 |
|---|---|---|---|
| 프로필 열람 무료 | 친친·한다리더 무료 (한계비용 ≈ 0) | 3일, 일 5명 캡 | 열람 시 `paid_views` 영속화 — 트라이얼 만료 후에도 재열람 무료 |
| 반값 묶음 | 110물감 묶음 50% 할인 (10,000→5,000원) | 가입+3일, 1회 한정 | 환불 시 `bonusPoints` 로 환급 (출금 차단), `halfPricePackageUsed` flag 영구 |
| 무료 소개 요청 | 1회, 동일 주선자 1회만 | 가입+7일 만료 | 거절 시 카운터 복원 X. 팁(성의 표시)은 별도 사용자 부담. 주선자 보상은 정상 발생 (플랫폼 부담) |
| 팔레트픽 첫 달 무료 | 30일 풀 액세스 | 계정당 1회 | 카드 등록 X, 자동 만료, `firstSubscriptionUsed` flag 영구 |

> 1 가입자 worst-case 손실 ~13,500원. 단일 결제 (29,900원 팔레트픽 1회) 만 발생해도 break-even + 잉여.

## §6 외부 송금 유도 금지 (ADR 0046)

주선자는 사용자에게 앱 외부 채널(계좌이체·간편송금·현금)을 통한 감사 표시를 유도해서는 안 됩니다.

**적발 시 페널티**:
- 누적 잔액(`paidPoints` + `bonusPoints`) **전액 몰수**
- **출금 자격 영구 박탈**
- 등급 0 회귀 (모든 성사 기록 reset)
- 계정 **영구 정지** (동일 명의/디바이스 재가입 차단)

사용자에게는 외부 송금에 대한 환불·보상 의무 없음. 위반 신고는 앱 내 "신고하기" → "외부 송금 유도" 카테고리. 확정 시 신고 사용자에게 **50 물감** 보상.

> 5단 가드 (약관·UX nudge·키워드 탐지·인센티브 정렬·익명 신고) 상세는 ADR 0046 참조.

### 1.3 PG / 결제 수단

- **PG**: Toss Payments (베타: test 키, 정식: live 키)
- 결제 수단: 신용/체크카드, 카카오페이, 네이버페이 (Toss 가 제공하는 범위 내)
- 영수증: Toss 발급 메일/문자 (사용자 가입 이메일/휴대폰 기준)

## 2. 환불 / 취소

> 모든 환불은 잔액(물감)으로 복귀. 매칭 흐름 자체엔 비용 X — 비용은 §1.1 잔액 차감에서 발생.

### 2.1 프로필 열람 (20 / 30 물감 — ADR 0044)

| 시나리오 | 환불 가능 여부 |
|---|---|
| 미사용 잔액 환불 (구매 후 7일 내) | ✅ 100% (paidPoints 만) |
| 열람 직후 (5분 내) | ✅ 100% 환불 (오결제 보호) |
| 열람 후 5분~24시간 | ❌ 환불 불가 (정보 소비 완료로 간주) |
| 24시간 이후 | ❌ 환불 불가 |
| 시스템 오류로 열람 못 했을 때 | ✅ 즉시 환불 + 1회 무료 열람 보상 |

### 2.2 소개 요청 (100 물감 — ADR 0044)

| 시나리오 | 환불 |
|---|---|
| 사용자 본인이 24시간 내 취소 (주선자 미응답) | ✅ 100% 환불 (100 물감 복귀) |
| 주선자 거절 | ✅ 자동 환불 (100 물감 복귀) — 매칭 시도가 출발조차 못 함 |
| 대상자 거절 | ❌ 환불 불가 — 주선자 검토·전달까지 가치 발생 (주선자 분배는 미지급) |
| 매칭 성사 | ❌ 환불 불가 (정상 사용). 주선자에게 등급별 분배 (15~40 물감) 즉시 적립 |
| 쿨타임 등 시스템 사유로 전송 실패 | ✅ 즉시 환불 |

### 2.3 성의 표시 (옵셔널 tip, 10~500 물감, ADR 0044)

- 송금 후 환불 **불가** — 감사 표시 성격, 강제 결제가 아니므로 사용자가 신중히 선택
- 잘못 보낸 경우 운영자 CS 통해 케이스별 검토

### 2.4 팔레트 Pick 패스 (월 구독)

| 시나리오 | 환불 가능 여부 |
|---|---|
| 결제 직후 (24시간 내, 사용 X) | ✅ 100% 환불 |
| 사용 시작 후 | ❌ 환불 불가 (남은 기간 사용 가능, 다음 결제 해지만) |
| 자동결제 해지 | ✅ 언제든 (남은 기간은 사용 가능) |

### 2.5 보너스 잔액 (`bonusPoints`)

- 환불 **불가** — 무료로 지급된 보너스 (가입 환영·친구 가입·마일스톤)
- 만료 도달 시 자동 소멸

### 2.6 주선자 물감 출금

- 출금 가능 잔액: `paidPoints` (정상 보상 + 팁 수령). `bonusPoints` 출금 X
- 최소 출금 50 물감 (= 5,000원). 일·월 한도는 RUNBOOK 참조
- 출금 신청 후 3 영업일 내 입금
- 출금 취소 불가 (입금 완료 후)
- 부정 매칭(허위 정보, 시스템 오용) 적발 시 회수

### 2.7 환불 요청 채널

- 베타: 운영자 이메일 (admin@palette.kr) 또는 앱 내 CS
- 정식: 앱 내 CS + 콜센터 (정식 출시 시 결정)

## 3. 광고 / 마케팅

### 3.1 베타 단계
- **광고 없음** — 사용자 경험 우선
- 푸시 알림: 매칭/주선/시스템 알림만, 마케팅 푸시 없음

### 3.2 정식 출시 후 (검토 중)
- 인앱 광고 도입은 신중 — 데이팅 앱 특성상 brand-safety / 사용자 신뢰 영향 큼
- 검토 가능 옵션:
  1. **B2B 광고만** (결혼식장, 데이트 코스, 선물 등 — 데이팅 친화)
  2. 광고 절대 없음, 결제로만 수익
  3. 프리미엄 구독 (월 정액으로 광고/제한 제거)
- → 정식 출시 전 ADR 동반 결정

### 3.3 사용자 동의
- 가입 시 `agreedMarketing` 동의 별도 (현재 코드: User.termsAgreement.agreedMarketing)
- 동의 안 한 사용자는 마케팅 푸시/메일 발송 안 함

## 4. 사용자 책임 / 한계

### 4.1 앱이 보장하는 것
- 1촌 인증된 지인 네트워크 기반 신뢰 신호 제공
- 매칭 시 주선자 + 수신자 양쪽 승인 (단방향 매칭 방지)

> ⚠️ **신뢰도 점수(trustScore) — 미구현/보류 (ADR 0026).** 과거 문서의 사진/영상 기반 신뢰도 가산 스킴(사진 +10, 영상 +50 등)은 **서버에 구현된 적이 없으며** 실제 사용자 값은 항상 0이었다. 개인 프로필에는 등급/점수를 두지 않기로 하여(색깔만 유효) 신뢰도 UI를 제거했다. 백엔드 필드는 보존하되, 향후 "매너 온도"(만남 후 상호평가) 방향으로 재설계 검토.

### 4.2 앱이 보장하지 않는 것
- 만남 후 결과 (관계 성립, 결혼 등)
- 사용자가 작성한 프로필 정보의 사실성 (앱은 본인인증 + 사진 분석만 책임)
- 사용자 간 분쟁의 중재 — 단 명백한 약관 위반은 운영자 개입
- 매칭의 성공 보장

### 4.3 약관 위반 / 제재
- 허위 정보 등록 (사진/직업/학력 등) → 경고 → 휴면 → 영구 차단
- 다른 사용자에 대한 폭언/괴롭힘 → 즉시 차단
- 매칭 외 목적 (영업/광고/홍보) → 즉시 차단
- 미성년자 가입 → 즉시 차단 (NICE 본인인증으로 거름)

## 5. 개인정보 / PII

### 5.1 수집 항목

| 항목 | 수집 시점 | 보관 | 제3자 제공 |
|---|---|---|---|
| 이름 (실명) | 회원가입 + NICE 본인인증 | 탈퇴 시 즉시 삭제 | NICE (인증 시), 매칭 완료된 상대에게 |
| 이메일 | 회원가입 | 탈퇴 시 즉시 삭제 | 없음 |
| 휴대폰 번호 | 회원가입 + 인증 | 탈퇴 시 즉시 삭제 | NCP SENS (SMS 발송 시), 매칭 완료된 상대에게 |
| 생년월일 | 회원가입 | 탈퇴 시 즉시 삭제 | 없음 (나이만 노출) |
| 사진 | 프로필 작성 | 탈퇴 시 즉시 삭제 | 없음 (S3 비공개 버킷 + presigned URL TTL 1h) |
| 직업/학력/지역 | 프로필 작성 | 탈퇴 시 즉시 삭제 | 다른 사용자에게 (카테고리 라벨만, 회사명 정확 표기 X) |
| 결제 정보 | 결제 시 | Toss 보관 (앱은 결제 키만) | Toss |
| 활동 로그 (열람/매칭/탈퇴 사유) | 사용 중 | 탈퇴 후 90일 보관 (분쟁 대비) | 없음 |

### 5.2 외부 시스템 제공
- **NICE 본인인증**: 이름 + 생년월일 + 휴대폰 (정식 출시 후 실키 발급 시 활성)
- **NCP SENS**: 휴대폰 번호 (SMS 발송 시 — 베타는 bypass)
- **AWS S3**: 사진 (비공개 버킷, presigned URL 만 노출)
- **OpenAI**: 사용자 답변 (AI 인터뷰 → 소개글 생성 시). 사용자 식별 정보(이름/연락처) 포함하지 않음
- **Toss Payments**: 결제 정보 (테스트는 미연동, 정식 출시 시 활성)
- **FCM**: 푸시 토큰 (디바이스 기준)

### 5.3 보관 기간 / 파기
- 탈퇴 즉시 삭제: 사진, 프로필 답변, 연락처
- 90일 보관: 매칭 요청/결과 (분쟁 대비), 결제 내역 (전자상거래법 5년 의무 — 별도)
- 법정 보관: 결제 5년, 본인확인 정보 5년 (전자상거래법)

## 6. 신고 / 차단

### 6.1 신고 사유
- 허위 프로필 (사진 도용, 가짜 정보)
- 폭언/괴롭힘/스토킹
- 영업/홍보/광고 목적
- 미성년자
- 기타 약관 위반

### 6.2 처리
- 신고 접수 후 운영자가 24시간 내 확인 (베타: 1-2일)
- 명백한 위반 → 즉시 차단
- 모호하면 양측 소명 요청 후 판단
- 차단된 계정의 매칭 이력은 상대방에게 안내 (피해 보호)

## 7. 결혼정보업 등록 / 법적 자격

### 7.1 현재 상태 — **베타 / 외부 조사 중**
- 결혼중개업 vs 결혼정보업 vs 데이팅 앱 의 법적 분류 차이 검토 중
- 한국 「결혼중개업의 관리에 관한 법률」 (결혼중개업법) 적용 여부 — 등록·신고제

### 7.2 결혼중개업법 핵심 (참고)
- **국내 결혼중개업**: 신고제 (시·군·구청)
- **국제 결혼중개업**: 등록제 (요건 강화)
- 회원 정보 진실성 의무, 분쟁 시 손해배상

### 7.3 데이팅 앱 vs 결혼중개업
- 일반 데이팅 앱 (만남 알선) — 결혼중개업법 비적용으로 운영 사례 많음
- "결혼"을 명시적으로 표방하면 결혼중개업으로 분류 가능
- **Palette 의 포지셔닝**: "지인 네트워크 기반 진지한 만남" — 결혼 명시는 피하되, 보수적으로 사업자 신고 검토 권장

- 결정되면 본 문서 + ADR 0008 (예정) 에 결과 기록
- **주선자 정산(현금 지급)·다단계·전금법·무현금 모델 검토는 ADR 0064 + BACKLOG `CM-*` 로 분리 추적.** §2.6 출금 모델은 그 결정 전까지 베타에서 비활성(무현금) 운영 권장.
