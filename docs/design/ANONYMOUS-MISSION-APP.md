# 익명 미션/쪽지 앱 — 제품 설계 브리프 (별도 서비스)

> **Palette 인앱 기능이 아님.** Palette PRD 범위 밖.  
> 영감: 과거 ‘돛단배’류 — 익명 쪽지/미션이 랜덤 전달되고, 답장·상호 인정 후 정체가 열리는 설렘 루프.  
> 작성: 2026-07-14 · 상태: 초안 (미착수)

---

## 1. 한 줄 정의

**익명의 누군가가 보낸 가벼운 미션/질문에 답하고, 서로 “괜찮았다”고 인정하면 그때 프로필이 열리는 설렘형 소셜 데이팅(또는 소셜) 앱.**

Palette와의 관계:

| | Palette | 본 앱 (가칭 미정) |
|--|---------|-------------------|
| 신뢰 축 | 지인망 · 사람/팔리 주선 | 익명 · 미션 수행 |
| 프로필 | 먼저 읽고 소개 | **나중에** 연다 |
| 톤 | 진중 · 신뢰 | 가볍 · 설렘 · 플레이 |
| 결정 | — | **별도 앱·별도 브랜드** |

크로스프로모·계정 연동은 **런칭 이후에만** 검토. 초기에는 완전 분리.

---

## 2. 왜 Palette에 넣지 않는가

1. **정체성 희석** — Palette 웨지는 “마당발/주선”. 익명 랜덤은 다른 카테고리.  
2. **안전·모더레이션** — 사진 미션(손·룩)은 신고·불쾌 콘텐츠 비용이 큼.  
3. **스토어/포지셔닝** — 한 앱에 두 약속이 있으면 “뭘 하는 앱인지” 흐려짐.  
4. **실험 속도** — 별도 앱이 토글·브랜드·정책 실험이 빠름.

---

## 3. 핵심 루프 (MVP)

```text
1. 내가 미션/질문 카드 작성 (프리셋 + 짧은 커스텀)
2. 시스템이 익명으로 다른 유저에게 전달 (매칭 규칙: 지역·시간대·성비 가드)
3. 수신자가 수행/답장 (텍스트 또는 사진 — MVP는 텍스트·칩 우선)
4. 보낸 사람이 “잘했어요 / 패스” 선택
5. 양쪽 모두 OK면 → 서로의 미니 프로필 unlock
6. (선택) 대화 요청 / 연락 교환
```

### 3.1 미션 예시 (톤)

- 텍스트: “어떤 운동 좋아하세요?”, “오늘 기분 한 단어는?”  
- 행동(텍스트 인증): “오늘 점심에 돈까스 먹기! 한 줄로 인증”  
- 사진(Phase 2): “손 사진”, “오늘의 룩” — **옵트인·강한 모더레이션 후**

### 3.2 설렘 설계 포인트

- unlock **전**에는 닉네임·얼굴·스펙 비공개 (또는 매우 제한)  
- OK는 **양방향** — 한쪽만 OK면 프로필 비공개 유지  
- 패스해도 페널티 최소화 (가벼운 앱)

---

## 4. 차별화 / 리스크

### 강점
- 프로필 피로 없는 **먼저 플레이**  
- 일일·수시 미션으로 **리텐션**에 유리  
- Palette와 겹치지 않는 젊은 바이럴 포텐셜

### 리스크와 가드
| 리스크 | 가드 |
|--------|------|
| 불쾌·성적 미션 | 프리셋 화이트리스트 + 커스텀 필터/신고 |
| 사진 악용 | Phase 2, face blur 옵션, 즉시 신고·차단 |
| 스토킹성 반복 매칭 | 동일 쌍 재매칭 쿨다운, 차단 영구 |
| 성비 불균형 | 큐 밸런싱, 수신 캡 |
| “데이팅앱 규제” | 연령·위치·안전 센터 기본 탑재 |

---

## 5. MVP 범위 (별도 앱)

**In**
- 가입(소셜/전화) · 최소 프로필(성별·연령대·한 줄)  
- 미션 보내기/받기 (텍스트·칩)  
- 양방향 OK → 프로필 unlock  
- 차단·신고  
- 일일 수신 캡

**Out (초기)**
- Palette 계정/물감 연동  
- 지인망·주선  
- 장소 집합 미션(“청계천에 모여라”) — 별도 검토, 본 MVP 비포함  
- 유료 아이템 — 런칭 후 (부스터·미션 슬롯 등)

---

## 6. 브랜드·이름

- Palette / 팔리 브랜드 **직접 사용 금지** (혼동 방지).  
- 가칭은 별도 네이밍 세션. 문서에서는 “익명 미션 앱”으로만 지칭.  
- 훗날 soft cross-promo 시: “같은 팀이 만들었어요” 수준.

---

## 7. Palette와의 경계 (고정)

| 가능 | 불가 |
|------|------|
| 별도 레포·별도 스토어 리스팅 | Palette 홈 탭에 익명 미션 삽입 |
| 나중에 선택적 딥링크 프로모 | Pick/주선 플로우에 익명 쪽지 섞기 |
| 학습한 UX 패턴(칩 미션)을 Palette 일일 질문에 참고 | 한 앱 두 아이콘 전략으로 Palette 리뷰 설명 오염 |

Palette 측 공식 기록: **ADR 0078 Non-goals**, `docs/PRD.md` §9 범위 밖.
