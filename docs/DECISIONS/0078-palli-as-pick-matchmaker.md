# ADR 0078 — 팔리 = 팔레트 Pick 주선자 (서사·UX)

- **상태**: Accepted (설계 채택 · 구현은 후속)
- **날짜**: 2026-07-14
- **결정자**: ys.choi
- **선행**: [[0030]] (Pick = AI 직접 소개), [[0072]] (콜드스타트 공개 풀), [[0073]] (팔리 페르소나 유지), [[0047]] (Pick 파이프라인)
- **비범위**: 장소 귀인 미션(상시), 익명 미션/쪽지(돛단배형) — 후자는 **별도 제품** (`docs/design/ANONYMOUS-MISSION-APP.md`)

---

## 1. 배경

팔레트 Pick은 이미 `POST /matchmaking/direct` 로 **사람 주선자 없이** 대상에게 소개가 전달된다 (ADR 0030).  
기술적으로는 “AI/시스템이 주선”이지만, UI는 여전히 **추천 카드 + 소개 요청**에 가깝고, 지인 주선과 대비되는 **주선자 페르소나**가 약하다.

팔리(마스코트)는 이미 AI 인터뷰·연애 리포트의 화자(ADR 0073)다.  
콜드스타트에서 “주선자가 없다”가 아니라 **“기본 주선자가 팔리”**가 되면, 웨지(마당발을 시스템으로 대체·증폭)와 언어가 맞는다.

---

## 2. 결정

### 2-1. 역할 분리

| 주선자 유형 | 언제 | 승인 | 보상 |
|-------------|------|------|------|
| **사람 주선자** | 지인(1~2촌) 경로 | 2단계 (주선자 → 수신자) | 명예 + 폐쇄형 크레딧 (ADR 0064) |
| **팔리 (시스템 주선자)** | 팔레트 Pick / degree=0 | 주선자 단계 **자동 승인** → 수신자만 응답 (현행 direct) | 없음 (플랫폼 서사) |

- 사람 주선자 통계·레벨·대시보드에 팔리 요청을 **집계하지 않음** (현행 placeholder `matchmakerId` 정책 유지 또는 `SYSTEM_PALLI` 센티널로 명시 — 구현 시 선택).
- 사용자에게는 **“팔리가 이어줘요”** 로 보이고, 내부적으로는 기존 direct 파이프라인을 재사용한다.

### 2-2. 제품 언어 (노출 카피 SoT)

| 위치 | 기존 톤 | 팔리 주선자 톤 |
|------|---------|----------------|
| 홈 탭 | 팔레트 Pick | **팔리가 보낸 추천** / 부제: 팔레트 Pick (탭 라벨은 Pick 유지 가능) |
| 카드 헤더 | AI 추천 · 궁합 N% | **팔리가 고른 분** · 궁합 N% |
| 추천 근거 | insight summary | **팔리의 한마디** (3줄 이내, 기존 insight 재사용) |
| 상세 CTA | 소개 요청하기 | **팔리에게 이어달라고 부탁하기** (또는 짧은 형태: **팔리로 소개 부탁**) |
| 상세 안내 | “공통 친구 없이 바로…” | **팔리가 주선자예요. 상대에게 바로 전달돼요.** |
| 수신 알림 | 팔레트 Pick / 시스템 | **팔리가 ○○님을 소개하고 싶어 해요** |
| 히스토리 | AI 직접 소개 | **팔리 주선** |

용어 규칙(GLOSSARY):  
- 사용자 행위는 계속 **소개 요청**.  
- **주선**은 역할어 — 팔리도 “주선자”로 불러도 된다 (시스템 주선자).

### 2-3. UX 흐름 (목표)

```text
홈 · 팔레트 Pick
  → 카드: 팔리 아바타 + “팔리가 고른 분” + 궁합/한줄
  → 프로필 상세 (degree=0)
  → CTA: 팔리에게 이어달라고 부탁
  → (물감/패스 정책 동일) POST /matchmaking/direct
  → 수신자: “팔리가 소개를 전했어요”
  → 수신자 승인 → 성사 (사람 주선과 동일 성사 상태머신)
```

**WYSIWYG:** 요청자·수신자·히스토리 모두 같은 “팔리 주선” 라벨.

### 2-4. 화면별 요구 (구현 체크리스트 — 코드 변경은 후속)

1. **MainFeed / AiSignal 카드** — 팔리 아바타·“팔리가 고른 분”·한줄 근거  
2. **ProfileDetailScreen (degree=0)** — CTA·안내 카피 교체; 사람 주선 모달 비노출 유지  
3. **알림 카피** — `MatchmakingApproved` 등 Pick 경로 문구  
4. **IntroductionHistory** — 주선자 표시를 “팔리”로  
5. **(선택) 팔리 미니 시트** — 부탁 전 “왜 이 분인지” 3 bullet 확인  

### 2-5. 매칭 품질 (병행 백로그, UX와 분리)

- 벡터 임베딩·상호 fit 고도화는 **품질 레이어** (Phase 3 / ADR 0047 후속).  
- 본 ADR은 **서사·UX 통일**이 범위. 품질 개선 없이도 카피/플로우 선반영 가능.

## 4. 성공 지표 (채택 후)

- Pick 탭 → 상세 → 소개 요청 전환율  
- Pick 경로 수신 수락률 (사람 주선과 비교)  
- 콜드스타트(친구 0~2) 유저의 D1/D7 리텐션  
- 정성: “주선자가 있는 느낌” 사용성 인터뷰

---

## 5. 영향

- **데이터 계약**: 필수 변경 없음 (direct API 재사용). 표시명·알림 카피·선택적 `matchmakerDisplayName=팔리`.  
- **정책/가격**: 변경 없음.  
- **PRD**: §1.2·§5.1·범위에 팔리 시스템 주선자 반영.
