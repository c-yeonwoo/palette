# 0039 — 결제 모델 정립: 두 종류 티켓 + 옵셔널 팁 + 월 구독

- **상태**: Accepted (베타 단계 — 가격 일부 미확정)
- **결정일**: 2026-06-07
- **결정자**: ys.choi

## 컨텍스트

기존 모델이 `열람권` 한 종류 + `포인트(P)`로 혼재해 사용자가 "내가 무엇을 사고 있는지" 직관이 약했음. 또 프로필 열람 3,000원·5,000원이 시장 대비 비싸 진입 장벽.

## 결정

### 1. 두 종류 티켓 + 옵셔널 팁 + 월 구독

| 결제 형태 | 용도 | 차감 단위 |
|---|---|---|
| **프로필 열람 티켓** | 프로필 상세 열람 | 1장 / 2장 (거리별) |
| **소개 요청 티켓** | 주선자에게 소개 요청 | 1장 |
| **성의 표시 포인트** | 주선자에게 옵셔널 tip | 1P 단위 (사용자 선택) |
| **팔레트 Pick 패스** | AI 추천 무제한 | 월 9,900원 |

티켓 = 사용 단위 명확("열람", "소개 요청"). 포인트 = 자유로운 금액 (tip).

### 2. 프로필 열람 티켓 가격 하향

| 거리 | Before | **After** |
|---|---|---|
| 1촌 (지인) | 무료 | 무료 |
| 친구의 친구 | 1장 (≈3,000원) | **1장 (≈1,000원)** |
| 한 다리 더 건너 | 1~2장 (≈5,000원) | **2장 (≈2,000원)** |

→ 진입 장벽 ↓, 회전율 ↑. 묶음 할인은 Phase 2.

### 3. 소개 요청 티켓 (신규 개념)

매칭 시도라는 진지한 액션을 별도 티켓으로 분리. 사용자는 "이번에 정말 소개받고 싶다"는 의지로 1장 차감. 거절/철회 시 환불 정책은 POLICY §2.

- **가격**: 약 3,000원 (Phase 2 확정 — 매칭 성사 시 주선자 보상 평균 ~300~600P 의 5~10배가 합리적).

### 4. 성의 표시 포인트 (옵셔널 tip)

매칭 시도 직전·성사 후 사용자가 "감사 표시"로 주선자에게 직접 송금. 강제 아님.

- 범위: 1,000~10,000P
- 주선자 등급 통계엔 미반영, 출금 가능 잔액에만 합산
- 매칭 알고리즘에 영향 X (이를 통한 우대 매칭 = 어뷰징 위험)

## 결과

- 결제 모델 4가지로 명확히 분리: 열람 티켓 / 소개 요청 티켓 / 팁 / 구독.
- 사용자 입장에서 "내가 뭘 사는가" 직관.
- 열람 가격 1/3 수준으로 하향 → 베타 후 유저 확보 시점 진입 장벽 ↓.

## 후속 (Phase 2)

- 소개 요청 티켓 가격 확정
- 묶음 판매(5장/10장) 할인 설계
- 환불 정책 §2 갱신 (소개 요청 거절 시 환불 정책)
- 백엔드 도메인: `ViewTicket`, `IntroRequestTicket`, `TipTransaction` 엔티티
- Toss 결제 연동 (지금은 paymentKey 존재만 체크하는 stub)

## 메모

- 기존 `열람권` 용어 → **프로필 열람 티켓** 으로 GLOSSARY/POLICY 통일.
