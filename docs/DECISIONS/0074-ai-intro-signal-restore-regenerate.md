# ADR 0074 — AI 소개글: 잃은 신호 복원 + "다른 느낌으로 다시"

- 상태: 채택
- 날짜: 2026-07-14
- 관련: [[0047]] (팔레트픽), 0056(다근거 색 판별), 0059(섹션 소개글), 0068(적응형 인터뷰)

## 배경

AI 색·소개글 생성은 서비스의 핵심 기능(온보딩 전환의 마지막이자 프로필의 얼굴)이다.
인터뷰를 스토리 5문항으로 축소(0068 후속, PR #245)하면서 "구조화 정보는 이미 생성
요청에 포함된다"는 전제를 달았으나, 실제 `POST /ai-profile/generate` 요청을 점검하니
두 가지 신호 누수가 있었다:

1. **이상형 칩이 '코드'로 전달** — `idealType.personalities` 등은 온보딩에서 `option.code`
   로 저장되는데(IdealTypeScreen), 프론트가 이를 **변환 없이** 그대로 보내 LLM 이
   `ADVENTUROUS` 같은 코드를 읽었다. 색·소개글 정확도 저하.
2. **관심사(interests) 미전달** — 라이프스타일 단계에서 수집해 미리보기엔 쓰면서도
   생성 요청엔 넣지 않아, 인터뷰 축소로 사라진 취향 신호가 완전히 유실됐다.

또한 사용자가 결과가 마음에 안 들 때, 답변은 그대로 두고 **AI 결과만 다시 받는** 경로가
없었다(전체 인터뷰 재작성뿐). 재생성은 이 기능에서 기대되는 기본 상호작용이다.

## 결정

**정확도 — 잃은 신호 복원**
- 프론트(`AIProfileEnhanceScreen`)가 생성 요청 전 이상형 칩을 `lblList`(onboarding
  options)로 **코드→한글 라벨** 변환해 전달. 관심사도 라벨로 변환해 신규 `interests`
  필드로 전달.
- 백엔드 `ProfileGenerationRequest.interests` + 프롬프트 `【관심사·취미】` 블록 추가
  (목록 나열 금지, 일상·소개글 소재로 녹이도록 지시).
- 인젝션 표면 없음: 전부 칩/enum 라벨(자유서술 아님) — 0068 원칙 유지.

**UX — 재생성**
- `ProfileGenerationRequest.variant`(nonce) 추가. 값이 다르면 입력 해시가 달라져
  기존 결과 캐시를 건너뛰고 새 LLM 호출 → temperature 0.8 로 다른 결과.
  프롬프트엔 노출하지 않음(해시에만 반영).
- "다른 느낌으로 다시 써줘" 버튼이 `variant` 를 증가시켜 재생성. 비용은 기존
  per-user 일 5회 rate-limit(0047 안전바)로 이미 상한. "답변부터 다시"는 부차 액션으로 격하.

## 근거 / 대안

- **직업(jobCategory)은 제외** — 온보딩 코드→enum→라벨 이중 매핑이 필요해 취약하고,
  성격·소개글 신호로서 관심사·이상형보다 약하다(인터뷰가 애초에 'job' 문항을 뺀 이유).
  프로필 사실(fact)로는 이미 저장·노출됨.
- **캐시 유지** — variant 없는 최초 생성은 여전히 캐시 hit(비용 0). 재생성만 우회.

## 영향

- 기존 캐시는 입력(이상형 라벨화 + interests 추가)이 바뀌며 자연 무효화 → 다음 생성부터 새 결과.
- 후방호환: `interests`/`variant` 는 선택 필드(기본값) — 구버전 클라이언트도 그대로 동작.
