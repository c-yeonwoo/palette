# ADR 0075 — 프로필 = 스펙시트에서 '서사'로 (visitor)

- 상태: 채택 (visitor 1차 — self 통합은 후속)
- 날짜: 2026-07-14
- 관련: [[0059]] (섹션 소개글), [[0074]] (AI 소개글 신호), 0047(팔레트픽)

## 배경

내 프로필(MyProfileScreen)과 남의 프로필(ProfileDetailScreen)이 **정보도 느낌도 달랐다**:
- 내 프로필 = 관리 대시보드(단일 히어로 + 썸네일 스트립 + 통계 + 완성도 링 + 탭)
- 남의 프로필 = 평가 카드(스와이프 캐러셀 + 색궁합 + 탭 + sticky 소개요청)

둘 다 **탭 스펙시트**라 사람이 '읽히지' 않았고, 인터뷰 답변은 탭 뒤에 폼처럼 묻혔다.
결정적으로 지인망 서비스의 핵심 자산인 **친구 보증·공통 친구**가 남의 프로필엔 서사로
드러나지 않았다.

## 결정 (visitor 1차)

`ProfileDetailScreen`을 탭 제거 → **하나의 스토리 스크롤**로:

1. **히어로** — 대표사진 풀블리드 + 이름 + 색 칩 + 색 설명 한 줄(첫인상).
2. **공감 먼저** — 공통 친구(mutualFriends) 콜아웃 + 색궁합 배너를 히어로 직후로.
3. **이야기** — 인터뷰 Q&A(hobby/charm/passion/happiness/motto)를 비트로, **사이사이
   추가 사진을 인터리브**(대표사진은 히어로에 사용). 골드스푼식 페이싱.
4. **친구 보증** — vouchCount 를 서사 말미 인용 카드로 (남의 프로필에 처음 노출).
5. **한눈에 보기** — 기본정보 CategoryCard + (기존) 관심사·색·심리를 서사 뒤로 강등.
   스펙이 사람을 앞서지 않게.
6. **이런 인연을 찾아요** — 기존 '이상형' 탭을 스크롤 하단으로 흡수(탭 제거).
   신고/차단·열람 게이팅·sticky 소개요청은 유지.

AI 소개글 프롬프트(0074 후속)에 **거주 생활권(residence)** 을 추가 — 일상 섹션에
동네 감각으로 가볍게 녹이도록(스펙 나열 금지). 온보딩엔 근무지가 없어 거주지만 전달.

## 후속

- **self 통합**: MyProfileScreen 을 같은 스토리 프레임(mode=self)으로 — 편집 어피던스 +
  완성도 넛지 오버레이. WYSIWYG(내가 만든 것 = 남이 보는 것). → 별도 PR.
- 공통 관심사/색/MBTI 하이라이트(뷰어 데이터 필요)·소개글 리드 문단 분리는 추가 개선 여지.

## 영향

- 데이터 계약 무변경(기존 프로필 응답 필드 + mutualFriends + vouchCount 재사용).
- 죽은 코드 정리: PhotoCarousel·InterviewAnswer·activeTab 제거.
