# 0042 — 단일 잔액 모델 ("물감", 1 물감 = 100원)

- **상태**: Accepted (베타) — ADR 0039 폐기
- **결정일**: 2026-06-10
- **결정자**: ys.choi

## 컨텍스트

ADR 0039(4축 결제 모델 — 열람 티켓·소개 요청 티켓·옵셔널 팁·구독)의 두 가지 문제:

1. **포인트 충전 루트 부재** — 사용자가 팁 보낼 포인트를 어디서 얻는지 모호
2. **통화 분리로 인한 멘탈 모델 혼란** — "티켓 둘 + 포인트 + 구독" 4종 구분이 사용자 입장에선 부담

추가로 주선자 보상도 포인트(P) — 사용처와 보상이 같은 단위로 묶이는 것이 자연스러움.

## 결정

### 1. 단일 잔액 모델

모든 결제는 **단일 잔액(물감)** 에서 차감. 티켓 개념 폐기.

**단위**: 사용자 노출 **"물감"** (Paint), `1 물감 = 100원 (KRW)`. 코드 변수는 `P` / `points` 유지(Paint 의미).
별풍선·카카오톡 초코 같은 친숙한 단위감 + 브랜드(팔레트) 정체성 강화.

**잔액은 두 갈래로 내부 관리** (사용자 인지엔 합산):
- `paidPoints` — 유료 충전, 만료 없음, 환불 가능 / 출금 가능
- `bonusPoints` — 가입 환영·친구 가입·마일스톤, `bonusExpiresAt` 만료

소비 순서: bonus → paid (사용자에 유리).

### 2. 가격표

| 사용처 | 차감 (물감) | 원화 환산 |
|---|---|---|
| 1촌(친구) 본인 프로필 열람 | 무료 | 무료 |
| 친구의 친구 프로필 열람 | **10 물감** | 1,000원 |
| 한 다리 더 건너 프로필 열람 | **20 물감** | 2,000원 |
| 소개 요청 1회 | **30 물감** | 3,000원 |
| 성의 표시 (옵셔널 팁) | **10~100 물감** (자유) | 1,000~10,000원 |

### 3. 충전 묶음

| 받는 물감 | 결제 금액 | 보너스 |
|---|---|---|
| 50 | 5,000원 | — |
| **110** | 10,000원 | +10 (10%↑) |
| **340** | 30,000원 | +40 (13%↑) |
| **575** | 50,000원 | +75 (15%↑) |

→ "보너스 +N 물감" 형태로 충전 시점에 가치 인식 ↑.

### 4. 보너스 정책 (ADR 0041 갱신)

| 종류 | 지급 | 유효기간 |
|---|---|---|
| 가입 환영 보너스 | **30 물감** (= 친친 열람 3 + 소개 요청 1 가치) | 7일 |
| 친구 가입 보너스 (양쪽) | **10 물감** | 14일 |

### 5. 주선자 보상 (ADR 0038 호환)

기준 보상 **20 물감** (= 2,000원) × 등급별 커미션%:

| 등급 | 커미션 | 매칭당 보상 |
|---|---|---|
| Lv.1 브론즈  | 15% | **3 물감** |
| Lv.2 실버    | 20% | **4 물감** |
| Lv.3 골드    | 25% | **5 물감** |
| Lv.4 플래티넘 | 30% | **6 물감** |
| Lv.5 다이아  | 40% | **8 물감** |

→ 모두 정수, 등급별 격차 명확.

### 6. 마일스톤 보너스 (ADR 0041 갱신)

| 누계 매칭 | 보너스 |
|---|---|
| 첫 매칭 | +5 물감 |
| 5건  | +20 물감 |
| 10건 | +50 물감 |
| 20건 | +100 물감 |
| 50건 | +250 물감 |
| 100건 | +500 물감 |
| 150건 (다이아 도달) | +1,000 물감 |

### 7. 출금 (주선자만)

`paidPoints` (정상 적립분 + 팁 수령) 만 출금 가능. `bonusPoints` (보너스성)는 출금 X.
최소 출금: **50 물감** (= 5,000원). 일일 한도·월 한도는 정책값 유지.

## 결과

- **사용자 인지**: "내 물감 120" 한 숫자만 보면 됨. 모든 사용처 자동 차감.
- **개발자 인지**: `BillingService.consume(amountP, reason)` 단일 메서드 (변수명 P 유지, Paint 의미).
- **회계**: `paidPoints` × 100 = 원화 매출. `bonusPoints` × 100 = 부채 (만료 시 소멸).

### 데이터 마이그레이션

베타 단계 결제 미연동 → 실데이터 거의 없음. 새 컬럼(`paid_points`, `bonus_points`) 추가하고 기존 `view/intro_ticket_count` 컬럼은 deprecated. ddl-auto=update 시 새 컬럼 자동 생성.

기존 베타 사용자 잔액은 일괄 신규 가입 보너스 30 물감으로 정상화 (admin 일회성 잡).
