# 0042 — 단일 잔액 모델 (P 단위, 1P = 10원)

- **상태**: Accepted (베타) — ADR 0039 폐기
- **결정일**: 2026-06-10
- **결정자**: ys.choi

## 컨텍스트

ADR 0039(4축 결제 모델 — 열람 티켓·소개 요청 티켓·옵셔널 팁·구독)의 두 가지 문제:

1. **포인트 충전 루트 부재** — 사용자가 팁 보낼 포인트를 어디서 얻는지 모호
2. **통화 분리로 인한 멘탈 모델 혼란** — "티켓 둘 + 포인트 + 구독" 4종 구분이 사용자 입장에선 부담

추가로 주선자 보상도 포인트(P) — 사용처와 보상이 같은 단위로 묶이는 것이 자연스러움.

## 결정

### 1. 단일 잔액 모델 (P)

모든 결제는 **단일 잔액(P)** 에서 차감. 티켓 개념 폐기.

**단위**: `1P = 10원 (KRW)`. 별풍선·카카오톡 초코 같은 친숙한 단위감으로 심리적 가격 부담↓.

**잔액은 두 갈래로 내부 관리** (사용자 인지엔 합산):
- `paidPoints` — 유료 충전, 만료 없음, 환불 가능 / 출금 가능
- `bonusPoints` — 가입 환영·친구 가입·마일스톤, `bonusExpiresAt` 만료

소비 순서: bonus → paid (사용자에 유리).

### 2. 가격표

| 사용처 | 차감 P | 원화 환산 |
|---|---|---|
| 1촌(친구) 본인 프로필 열람 | 무료 | 무료 |
| 친구의 친구 프로필 열람 | **100P** | 1,000원 |
| 한 다리 더 건너 프로필 열람 | **200P** | 2,000원 |
| 소개 요청 1회 | **300P** | 3,000원 |
| 성의 표시 (옵셔널 팁) | **100~1,000P** (자유) | 1,000~10,000원 |

### 3. 충전 묶음

| 받는 잔액 | 결제 금액 | 보너스 |
|---|---|---|
| 500P | 5,000원 | — |
| **1,100P** | 10,000원 | +100P (10%↑) |
| **3,400P** | 30,000원 | +400P (13%↑) |
| **5,750P** | 50,000원 | +750P (15%↑) |

→ "보너스 +N P" 형태로 충전 시점에 가치 인식 ↑. 카카오톡 초코·게임 머니 익숙 모델.

### 4. 보너스 정책 (ADR 0041 갱신)

| 종류 | 지급 P | 유효기간 |
|---|---|---|
| 가입 환영 보너스 | **300P** (= 친친 열람 3장 + 소개 요청 1장 가치) | 7일 |
| 친구 가입 보너스 (양쪽) | **100P** | 14일 |

### 5. 주선자 보상 (ADR 0038 호환)

기준 보상 **200P** (= 2,000원) × 등급별 커미션%:

| 등급 | 커미션 | 매칭당 보상 |
|---|---|---|
| Lv.1 브론즈  | 15% | **30P** |
| Lv.2 실버    | 20% | **40P** |
| Lv.3 골드    | 25% | **50P** |
| Lv.4 플래티넘 | 30% | **60P** |
| Lv.5 다이아  | 40% | **80P** |

→ 모두 정수 P, 등급별 격차 명확.

### 6. 마일스톤 보너스 (ADR 0041 갱신)

| 누계 매칭 | 보너스 |
|---|---|
| 첫 매칭 | +50P |
| 5건  | +200P |
| 10건 | +500P |
| 20건 | +1,000P |
| 50건 | +2,500P |
| 100건 | +5,000P |
| 150건 (다이아 도달) | +10,000P |

### 7. 출금 (주선자만)

`paidPoints` (정상 적립분 + 팁 수령) 만 출금 가능. `bonusPoints` (보너스성)는 출금 X.
최소 출금: **500P** (= 5,000원). 일일 한도·월 한도는 정책값 유지 (단위만 P로 환산).

## 결과

- **사용자 인지**: "잔액 1,200P" 한 숫자만 보면 됨. 모든 사용처 자동 차감.
- **개발자 인지**: `BillingService.consume(amountP, reason)` 단일 메서드.
- **회계**: `paidPoints` × 10 = 원화 매출. `bonusPoints` × 10 = 부채 (만료 시 소멸).

### 데이터 마이그레이션

베타 단계 결제 미연동 → 실데이터 거의 없음. 새 컬럼(`paid_points`, `bonus_points`) 추가하고 기존 `view/intro_ticket_count` 컬럼은 deprecated. ddl-auto=update 시 새 컬럼 자동 생성.

기존 베타 사용자 잔액은 일괄 신규 가입 보너스 300P 로 정상화 (admin 일회성 잡).

## 메모

- **명칭**: 사용자 노출은 "P" 또는 "잔액". "포인트" 단어는 게임 인식 회피 위해 자제.
- "팔레트 P" 또는 "팔레트 머니" 같은 브랜드명 검토 가능 (마케팅 결정).
- ADR 0024 (열람권 도입), 0038 (커미션·승급), 0041 (체험·보너스) 는 이번 ADR 단위 변경에 맞춰 본문 그대로 유효 (단 단위만 P로).
