# 0070 — 프리미엄 "팔레트 분석 리포트" (풀 리포트 + 매력 육각형)

- **상태**: Accepted
- **작성일**: 2026-06-21
- **작성자**: ys.choi
- **선행**: ADR 0056(다근거 색 분석), 0059(소개글 섹션), 0067(코랄 브랜드·anti-spec), 0069(색 리포트 잠금)

---

## 1. 문제 / 의도
ADR 0069 의 "심층 리포트"는 영속된 색 필드(성향·이상형·강점)를 블러로 가린 수준이라, **돈 내고 볼 만한 깊이**가 부족했다.
유저 요청: "당신의 색·결·매력·어울리는 인연·장단점·목표 + 사주로 운/연애운까지 엮은 **연애운 리포트**"와
**육각형 분포 분석**까지. 잠금 범위도 한 섹션이 아니라 **풀 리포트 전체**.

## 2. 결정
ColorDetailScreen 을 **무료 티저 + 유료 풀 리포트**로 확장. 잠금(물감 5 / 광고)을 풀면 아래 전체 공개.

- **무료(훅)**: 색 타입·키워드·"AI가 이 색을 고른 이유"(reasoning).
- **유료 풀 리포트** (`GET /api/v1/color-report/full`, 해제 시에만 200·미해제 403):
  ① 당신의 결(essence) ② **매력 육각형**(hexagon + 해석) ③ 당신의 매력+강점 ④ 성장 포인트 ⑤ 어울리는 인연
  ⑥ 연애 스타일 ⑦ **연애운(사주)** ⑧ 팔리의 조언. 한 줄 태그라인은 히어로에 노출.
- **매력 육각형 6축**: 온기·활력·깊이·감성·진정성·균형 (0~100). 외모·학력·자산 같은 **스펙 축을 의도적으로 배제**(ADR 0067 anti-spec).
  완벽한 정육각형이 아니라 **비대칭 모양 자체가 그 사람의 색**이라는 메시지(hexagonComment·UI 카피로 강조).
- **연애운(사주)**: 오행을 "**재미로 보는 보조 해석**"으로만 — 단정·운명론 금지, 마지막은 "운명이 아니라 당신의 매력이 인연을 끌어당긴다"로 환원.

### 구현
- LLM: `OpenAIService.generateColorReport(ColorReportContext)` — 전용 시스템 프롬프트(`COLOR_REPORT_SYSTEM_PROMPT`).
  안전바는 generate/adaptive 와 동일(키없음·실패 → `stubColorReport` graceful, 절대 500 X). usage log purpose=`color_report`.
  stub 도 비대칭 육각형 + 8섹션 + 연애운 재미 caveat 를 합성(로컬 검증용).
- 캐시: `color_report_cache`(유저당 1행, `input_hash` 로 프로필 변경 감지 → 자동 갱신). 마이그레이션 §26(`CREATE TABLE IF NOT EXISTS ... ENGINE=InnoDB`). 같은 프로필이면 LLM 재호출 0.
- 컨트롤러: `ColorReportController` 에 `/full` 추가 — 영속 프로필(인터뷰 답변·이상형·MBTI)+사주(`SajuService`)로 컨텍스트 구성, 이상형 칩은 `field_options` 로 한글 라벨 해석(인젝션 표면 없음). 잠금/해제(`/`, `/unlock`)는 ADR 0069 그대로.
- 프론트: `PaletteRadar`(외부 차트 의존 없는 손제작 SVG 6축 레이더) + 풀 리포트 섹션 렌더. 잠금 시 블러된 목차 위 "연애 운명 리포트" 해제 CTA.

## 3. 트레이드오프 / 한계
- **사주**는 오락 요소 — 재미 caveat 를 프롬프트·UI 양쪽에 박아 단정/미신 리스크 차단.
- **육각형 점수의 근거**: LLM 이 답변·MBTI·사주로 산출 — 정량 모델이 아니라 해석. "최고-최저 25+ 차이" 가이드로 평탄화 방지.
- **로컬은 stub** (dummy 키) — 진짜 소울풀한 리포트는 prod(OPENAI_API_KEY)에서만. stub 은 그럴듯하지만 템플릿.
- **광고 무료 해제**: 여전히 placeholder(준비 중) — AdMob/Capacitor 후속.
- **상대 리포트 유료 열람**: 보류(ADR 0069 와 동일) — 주인 옵트인 설계 필요.
