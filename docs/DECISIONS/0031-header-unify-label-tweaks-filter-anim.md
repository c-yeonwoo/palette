# 0031 — 헤더 통일/개선 + 마이페이지 라벨 정리 + 필터 애니메이션

- **상태**: Accepted
- **결정일**: 2026-06-07
- **결정자**: ys.choi

## 컨텍스트

마이페이지·홈 카피/헤더 디테일 피드백.

## 결정

1. **상단바(헤더) 통일·개선** — 전 화면 sticky 헤더 높이 `h-14`→`h-16`, 타이틀 `text-base`→`text-lg`(7개 화면). 좁고 밋밋하던 헤더에 여백·존재감 부여, 페이지 간 통일.
2. **마이페이지 완성도 뱃지** — "완성도 N%" → **"프로필 완성도 N%"**.
3. **내 활동 메뉴** — "친구 연결" → **"지인 관리"** (subtitle "초대 코드 & 지인 검색"). FriendConnectScreen 헤더와 일치.
4. **공개 설정 토글 라벨** — "소개 받기" → **"주선 요청 받기"** (label/desc/토스트/확인 다이얼로그). `isAcceptingMatches` 동일. GLOSSARY Layer 3 갱신.
   - 참고: ADR 0024가 "주선받기"→"소개 받기"로 바꿨으나, 수신자 토글은 "주선 요청 받기"가 더 명확하다는 판단으로 재조정. 발신자 행위 용어("소개 요청")는 유지 — 같은 요청을 발신=소개 요청 / 수신 토글=주선 요청 받기로 표기(사용자 선호).
5. **카드 커버 가이드 문구 제거** — "탭해서 열기"/"여는 중…" 칩 삭제. 물감 커버만 노출(미니멀).
6. **소개 페이지 필터 진입 애니메이션** — 우측 슬라이드(`slide-in-right` 0.28s) + 백드롭 페이드(`fade-in`). theme.css에 keyframe·유틸 추가.

## 결과

- 헤더 통일·개선, 마이페이지 카피 정리, 필터 슬라이드 인.
- vite build / vitest(40) 통과.

## 메모

- 토글 라벨 "주선 요청 받기"는 GLOSSARY의 "주선=주선자 역할어" 원칙과 미세 긴장 — 수신 토글 한정 예외로 기록. 추후 통일 필요 시 재검토.
