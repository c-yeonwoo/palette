# 0072 — 콜드스타트 공개 발견 풀 + 시군구 거리 (순차적 차별화)

- **상태**: Accepted
- **작성일**: 2026-07-12
- **작성자**: ys.choi
- **선행**: ADR 0047(팔레트 Pick 매칭 파이프라인 §B), 0064(무현금 보상), 0071(내친소 — 콜드스타트 Phase 2 대응), 0022(매칭 받기 설정), 0023(차단 격리), 0035(상세 공개 범위)
- **대체**: 프론트 "데모 풀 시드 유저 12명" 하드코딩 패치(임시 콜드스타트) → 엔진 레벨 폴백으로 대체. ADR 0064 의 "현금 90/10·커미션" 잔재를 **명예 + 폐쇄형 크레딧**으로 확정(사업계획서 불일치 해소).

---

## 1. 문제 / 의도

팔레트는 **양면 마켓플레이스**다 — 희소한 주선자(공급)와 솔로(수요). 근데 신뢰 차별화의 핵심인 지인 주선은 **네트워크가 형성된 뒤에야** 켜진다. 신규 유저(친구 0명)는 시작부터 빈 풀을 본다.

코드 확인 결과, 콜드스타트가 실제로 안 뚫린다:
- `CandidatePoolService.build()` ([CandidatePoolService.kt:74](../../src/main/kotlin/kr/ai/palette/palettepick/application/CandidatePoolService.kt)) 가 **2촌(친구의 친구)에서만** 후보를 시작한다. 친구 0명 → 1촌 없음 → 2촌 없음 → 팔레트픽이 아무것도 반환 못 함(`recommend()` early-return empty).
- 거리·지역 차원이 아예 없다.
- 실제로 콜드스타트를 때우는 건 팔레트픽 엔진이 아니라 **프론트의 시드 유저 12명 하드코딩**이다. 즉 해자로 설계한 엔진과, 콜드스타트를 막는 것이 분리돼 있다.

**전략 결정(창업자):** 팔레트픽을 **콜드스타트 가치 엔진**으로 삼는다 — 지인 네트워크가 없어도 수도권·거리순·이상형 매칭된 실제 프로필을 day-1에 보여줘 쓸 이유를 만든다. 지인 주선은 그 위에 얹히는 신뢰 레이어로, 네트워크가 두꺼워지며 활성화된다. = **순차적 차별화(sequenced differentiation).**

**리스크(명시):** 콜드스타트 구간에선 팔레트픽 = 사실상 AI 데이팅앱(글램·아만다와 동일 링, 색 스킨만 다름). 주선 차별화는 나중에야 켜진다. **네트워크가 절대 안 두꺼워지면 영원히 "색 입힌 약한 글램"에 머문다.** → 완화책은 §2-4 densification 컨베이어(콜드스타트 사용 자체가 친구 그래프 엣지를 만들게 설계).

## 2. 결정

### 2-1. 후보 풀 폴백 티어 (핵심)
`CandidatePoolService.build()` 를 2단 구조로:

1. **지인 티어 (신뢰 우선)** — 현행 2촌 풀. 항상 앞 순위.
2. **공개 발견 폴백** — 지인 티어가 `MIN_ACQUAINTANCE_POOL`(초기 5) 미만이면, 공개 풀로 보강:
   - 조건: `publicDiscoverable=true`(§2-3) + `canReceiveMatches()`(ADR 0022) + 이성 + 자신/차단(유저·운영자)/숨김/60일 이력 제외(ADR 0009·0011·0023) + 수도권(§2-6)
   - **시군구 중심좌표 거리 오름차순 정렬 → maxSize(50)까지 채움.**
   - 지인 후보가 먼저, 공개 후보가 뒤. 중복 제거.

스코어링 파이프라인(`MutualFitScoringService`, `PalettePickScore` 4축)은 **불변** — 풀만 채워지면 그대로 동작. 지인/공개 출처 태깅은 추천 응답 메타로 노출(프론트가 "공통 지인 N명" vs "가까운 거리" 구분 표기).

### 2-2. 거리 = 풀 경계 + 정렬 키 (스코어 축 아님, v1)
거리는 **공개 폴백 풀의 지리적 경계 + 정렬**에만 쓴다. `PalettePickScore` 의 5번째 축으로 넣지 않는다.
- 근거: (a) 단순·저위험 — 4축 가중치 재튜닝 불필요. (b) 콜드스타트 유저는 임베딩이 희소(AI 인터뷰 전)해 4축 품질 신호가 약함 → 물리적 거리가 더 신뢰할 만한 초기 신호. "가까운 사람들 중 이상형 적합도 높은 순"이라는 자연스러운 결합.
- 미래 튜닝 레버: 거리 스코어 축(option b)은 데이터 쌓인 뒤 A/B 대상(ADR 0047 §후속).

**시군구 중심좌표 Haversine 근사** — 정식 위/경도(기기 위치 권한) 안 받음. 초기 마찰 최소화 + 시/군/구 단위면 데이팅 거리 정렬에 충분. `sido+sigungu → (lat,lng)` 정적 lookup(수도권 서울·경기·인천). 좌표 없는 시군구는 폴백 풀에서 거리 unknown → 후순위.

### 2-3. `publicDiscoverable` 설정 (opt-in, 기본 ON)
`ProfileSettings` 에 `publicDiscoverable: Boolean = true` 추가.
- **의미**: "지인 네트워크 밖의 사용자에게도 콜드스타트 발견 풀에서 내가 노출될 수 있음." `isAcceptingMatches`(소개/주선을 받을지 자체)와 **별개** — 지인 매칭은 받되 공개 노출은 끄는 조합 허용.
- **기본 ON** — 초기 풀 유동성이 서비스 생존선. 단 (a) 가입/설정에 명확 고지, (b) 마이페이지 "공개 범위"에서 언제든 OFF, (c) OFF 시 지인 티어(2촌)에는 계속 노출(신뢰망은 유지).
- 마이그레이션: `detailsVisibleToFriends` 선례대로 `schema-mysql-migrations.sql` 멱등 ALTER(`public_discoverable BOOLEAN NOT NULL DEFAULT TRUE`). dev(H2)는 Hibernate DDL.

### 2-4. Densification 컨베이어 ("약한 글램" 리스크 완화)
콜드스타트 사용이 친구 그래프 엣지를 만들어내게:
- 공개 후보 카드에 **공통 지인 힌트**("혹시 아는 사이? / 공통 지인 N명") — 그래프 형성 유도.
- **클러스터 초대** — 초대를 학교/회사 단위로 묶어 국소 2촌 밀도 형성(발견은 광역·거리순, 신뢰 밀도는 클러스터).
- 초대 보너스 강화(기존 ADR 0048).
- (구현은 단계적 — 힌트/클러스터 초대는 폴백 풀 착지 후 후속 스프린트. BACKLOG 추적.)

### 2-5. 주선자 보상 = 명예 + 폐쇄형 크레딧 (확정)
ADR 0064 의 무현금 방향을 확정: 주선자는 **현금이 아니라 (a) 명예(리그·랭킹·"내가 이어준 커플" 스토리) + (b) 폐쇄형 크레딧(물감/열람권/구독 크레딧)** 으로 보상. 현금 출금·90/10 팁·커미션은 채택 안 함(사업계획서의 현금 분배 서술은 이 ADR로 폐기·수정).
- 근거: 컴플라이언스(전자금융·세무·다단계) 회피 + 재사용 루프(크레딧이 다시 플랫폼 내 소비로) + 공급 동기 유지.
- **⚠️ 미해결 정책 충돌 (CS-009)**: 이 절의 (b) 폐쇄형 **크레딧(물감)** 은 현행 `POLICY.md §1.2`(ADR 0064: "주선자는 금전·**물감** 보상을 받지 않으며 … 명예만")·`MatchmakingService.acceptByTarget`의 "마일스톤 물감 보상 비활성" 주석과 **정면 충돌**. 즉 ADR 0064 는 물감 보상까지 껐고, 본 ADR 0072 는 폐쇄형 물감 크레딧을 되살리려 함. 어느 쪽이 유효한지(주선자에게 성사 시 물감 지급 O/X, 지급 시 액수)는 **대표 결정 대기** — 결정 전까지 CS-009(크레딧 지급 구현) 보류. 결정 시 POLICY §1.2 ↔ ADR 0064/0072 동기화 필요.

### 2-6. 지역 스코프 (v1)
발견 풀 시드는 **수도권(서울·경기·인천)**. 대학/직군 게이트는 없음(직군 무관). 거리순 정렬이 지리를 처리하므로 단일 커뮤니티 하드 게이트 불필요. 수도권 밖은 v1 폴백 풀에서 제외(지인 티어에는 무관하게 노출).

## 3. 결과 / 트레이드오프

**+**
- 콜드스타트가 **엔진**으로 실제 동작(하드코딩 12명 대체). day-1 가치.
- 거리 선호(#2 결정) + 이상형 매칭 결합. 프라이버시 존중 기본값(기본 ON + 쉬운 OFF + 지인망 유지).
- 지인 후보 우선 → 유동성이 생겨도 신뢰 차별화 우선순위 보존.

**−/리스크**
- 콜드스타트 구간 = AI 데이팅앱 경쟁(레드오션). → §2-4 컨베이어 + 주선 레이어로 완화. densification 지표를 북극성으로 관측 필수.
- 시군구 중심좌표는 city-level 근사(동 단위 아님) — 데이팅 거리엔 충분하다 판단.
- 공개 풀은 opt-in 유동성에 의존 → 기본 ON으로 완화하되 프라이버시 인식 관리(고지·off).
- 수도권 편중(초기 의도된 스코프).

**북극성 지표 전환**: "가입·매칭" → **네트워크 밀도(유저당 평균 2촌 풀 크기)** + 콜드스타트 카드 오픈률 + 공개풀→지인엣지 전환율. 어드민 팔레트픽 메트릭에 출처(지인/공개) 분해 추가.

| 레이어 | 변경 |
|---|---|
| 도메인 | `ProfileSettings.publicDiscoverable`(기본 true) + 헬퍼 |

| 추천 | 응답 메타에 출처(ACQUAINTANCE/PUBLIC) + 거리 노출(프론트 표기용) |
| 프론트 | 콜드스타트 팔레트픽 실동작(데모12 대체) + 거리 배지 + "공개 범위" 토글 + 공통 지인 힌트 |

**단계적 배포**: (1) geo 유틸 + 설정 필드 → (2) 후보 풀 폴백 + 스코어 파이프라인 검증 → (3) 프론트 실동작 + 토글 → (4) densification 힌트(후속).
