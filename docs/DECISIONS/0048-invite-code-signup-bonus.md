# 0048 — 초대 코드 가입 보너스

- **상태**: Accepted (베타)
- **결정일**: 2026-06-12
- **결정자**: ys.choi
- **선행**: ADR 0041 (온보딩·리텐션 보너스), 0042 (단일 잔액 모델), 0044 (가격 v2)

## 컨텍스트

기존 가입 흐름:
1. **베타 코드** 입력 → 게이트 통과 (계정과 무관, 단순 게이팅)
2. **가입 환영 보너스** 60 물감 (7일 만료) 자동 지급
3. **친구 가입 보너스** 10 물감 (14일) — 친구 수락 시점 (가입 후)

문제:
- 지인이 초대해서 가입한 경우와 베타 코드만 알고 익명 가입한 경우의 차별화 없음.
- 신뢰의 시그널 (지인 네트워크) 이 보상으로 연결 안 됨.
- 가입 후 친구 요청을 별도로 거쳐야 친구 관계 형성 → 마찰점.

## 결정

가입 화면에 **초대 코드** 옵션 필드 추가. 유효 코드로 가입 시:

1. **양쪽에 100 물감 보너스** (30일 만료, 보너스 잔액 — 출금 X)
2. **자동 1촌 친구 관계** 형성 (초대자 ↔ 신규 가입자, status=ACCEPTED)
3. **1회용 코드 소비** (`inviteCodes` row delete)

### 금액 결정 근거

| 비교 기준 | 물감 | 환산 |
|---|---|---|
| 초대 코드 보너스 (양쪽) | **100** | 10,000원 / 친친 5명 열람 / 소개 요청 1회 |
| 가입 환영 (개인) | 60 | 6,000원 / 친친 3명 |
| 친구 수락 보너스 (양쪽) | 10 | 1,000원 / 친친 0.5명 |
| 소개 요청 1회 | 100 | 10,000원 |

- 친구 수락 보너스(10) 대비 **10배** → "초대 코드 = 강한 신뢰 시그널" 차별화
- 소개 요청(100) 과 동일 가치 → "초대 = 디지털 소개의 1회 가치"
- 신규 가입자 최대 시작 잔액 = 환영(60) + 초대(100) = **160 물감** → 친친 8명 무료 열람 → 충분한 가치 학습
- 30일 만료 → retention KPI 발생 + 빠른 가치 사용 유도
- 보너스 잔액 → 출금 불가 → 봇 어뷰징 방어

### 어뷰징 가드

- 초대 코드는 1회용 (사용 후 delete)
- 자기 자신의 코드 사용 차단 (백엔드 `inviterId == newUserId` 체크)
- 코드 만료(24시간) 시 자동 삭제
- 보너스 잔액 → 출금 불가 → 가짜 계정으로 인출 시도 무력화
- 향후: 동일 IP/디바이스/전화번호 다중 가입 자동 플래그 (별도 PR)

## 구현

### 백엔드

- `WelcomeBonusService.grantInviteCodeReward(inviterUserId, newUserId)` 추가
  - 설정: `app.bonus.invite-code-points=100` / `app.bonus.invite-code-valid-days=30`
- `EmailSignupRequest.inviteCode: String?` 옵셔널 필드 추가
- `EmailAuthController.signup()` 가 가입 후 `processInviteCode(inviteCode, savedUser.id)` 호출:
  - 코드 lookup → 만료/자기사용 가드 → 자동 친구 생성 → 양쪽 보너스 적립 → 코드 삭제
  - 실패 시 try/catch swallow + 로그 (가입 자체는 무조건 성공)
- 기존 `FriendshipController.joinByInviteCode` (가입 후 코드 사용) 는 그대로 유지 — 이중 진입 경로 제공

### 프론트엔드

- `EmailSignupScreen` 에 초대 코드 입력 필드 (옵션) 추가
  - 모노스페이스 + uppercase + maxLength 10
  - 안내: "지인에게 받은 코드를 입력하면 양쪽에 100 물감 + 자동 친구"
- 가입 성공 토스트: 코드 유효 시 "보너스 100 물감 받았어요 🎨" 추가

### SoT 갱신

- `docs/PRD.md` §7.2 보너스/체험 표에 초대 코드 행 추가
- `WelcomeBonusService.kt` 의 KDoc 에 100/30 정책 명시

## 결과

- 지인 추천으로 가입한 사람을 시스템적으로 우대 → 핵심 가치(지인 네트워크 신뢰) 강화
- 초대자에게도 보상 → 자발적 초대 확산 동기
- 가입과 동시에 1촌 관계 형성 → 첫 매칭 학습까지의 마찰 ↓
- 가입 환영 + 초대 보너스 = 160 물감 시작 → 친친 8명 무료 열람 → 가치 학습 충분

## 후속

- 초대 코드 사용 통계 (admin metrics) — 코드 발급 수 vs 가입 전환율
- 가입 후 별도 친구 수락 보너스(10P, ADR 0041)와의 정합성 — 같은 흐름 중복 보상 X (초대 코드는 코드만 소비, 친구 수락 보너스는 별도 흐름)
- 어뷰징 신호 누적 시 IP/디바이스 단위 가입 제한 ADR 별도 검토
