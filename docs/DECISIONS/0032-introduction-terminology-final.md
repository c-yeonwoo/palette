# 0032 — 사용자 행위 용어 "소개"로 최종 통일 (주선=역할어)

- **상태**: Accepted (ADR 0031의 "주선 요청 받기" 토글 예외를 대체, ADR 0024 재확정)
- **결정일**: 2026-06-07
- **결정자**: ys.choi

## 컨텍스트

ADR 0024가 사용자 행위를 "소개"로 정리했으나, 이후 수신 토글이 "주선 요청 받기"로 바뀌며 발신("소개 요청")과 수신("주선 요청 받기")이 갈렸다. 같은 요청 객체가 보낼 땐 소개·받을 땐 주선으로 표기돼 혼란 → 한쪽으로 통일 필요.

## 결정

**사용자 행위 용어를 "소개"로 통일한다. 주선은 주선자 역할·서비스 층 전용으로 유지.**

두 단어는 경쟁 동의어가 아니라 **층위**가 다르다:
- **주선 = 역할·서비스 층** — 주선자, 주선 대시보드, 주선 활동, 주선 성사. (신뢰·책임·커미션 차별점이 여기 담김)
- **소개 = 사용자 경험 층** — 소개 요청하기 / 소개 요청 받기 / 소개팅. (이용자가 주고받는 행위·사건)

→ "주선자가 **소개**해준다"로 자연스럽게 공존. 한국어 실사용(결혼정보회사 "전문 주선자가 소개해드립니다", 친구가 "내가 소개해줄게")과 일치하고, 소개팅 문화의 따뜻·친숙한 톤을 얻는다. 전부 "주선"으로 밀면 단어 패밀리는 일치하나 사용자 입장에서 사무적이고 "주선 받기"가 어색.

### 근거
- **소개팅 문화 적합** — 팔레트는 본질이 지인 소개팅. "소개받다/소개해줘"가 가장 자연스러운 멘탈 모델.
- **사용자 중심** — 유저 경험은 "내 색과 맞는 사람을 소개받는다". 주선은 운영/사업자 관점.
- **차별점 유지** — 신뢰·진지함은 "주선자"(역할)에서 그대로 전달되므로, 행위어를 소개로 둬도 브랜드가 희석되지 않음.

## 적용

- 수신 토글 "주선 요청 받기" → **"소개 요청 받기"** (MyPageScreen: 라벨/설명/토스트/다이얼로그).
- 발신은 기존대로 "소개 요청하기" / 팔레트 Pick 직접 "소개 요청하기".
- 주선자 관련어(주선자, 주선 대시보드, 주선 활동 등)는 **전부 유지**.
- GLOSSARY Layer 3·금지단어·변경 이력 갱신.

## 결과

- 발신·수신 모두 "소개 요청"으로 일관. 역할어 "주선자"와 층위로 공존.
