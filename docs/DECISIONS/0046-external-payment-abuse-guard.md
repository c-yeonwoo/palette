# 0046 — 외부 송금 어뷰징 가드 (5단)

- **상태**: Accepted (베타)
- **결정일**: 2026-06-10
- **결정자**: ys.choi
- **선행**: ADR 0044 (가격 v2 — 팁 90/10 분배)

## 컨텍스트

ADR 0044 에서 팁 90% 주선자 / 10% 플랫폼 수수료 도입. 주선자가 사용자에게 **외부 송금 유도** 시 플랫폼 수수료 회피 가능.

예: 주선자가 "앱 통하지 말고 내 계좌로 5만원 직접 송금해" 요구
→ 사용자 5만원 송금 (분쟁 시 보호 X)
→ 주선자 5만원 전액 수령 (정상 시 4.5만원)
→ 플랫폼 매출 0 + 사용자 위험 노출

**100% 차단 불가능** (전화·외부 메신저). 단 "외부 송금 기대수익 < 기대손실" 비대칭을 만들면 합리적 주선자는 앱 내 송금 선택.

## 결정 — 5단 가드

### A. 약관·페널티 (가장 강한 deterrent)

`docs/POLICY.md §6` 신설 + 약관 §6 추가:

> **§6 외부 송금 유도 금지**
> 주선자는 사용자에게 앱 외부 채널(계좌이체·간편송금·현금)을 통한 감사 표시를 유도해서는 안 됩니다.
>
> **적발 시 페널티**:
> - 누적 잔액(`paidPoints` + `bonusPoints`) **전액 몰수**
> - **출금 자격 영구 박탈**
> - 등급 0 회귀 (모든 성사 기록 reset)
> - 계정 **영구 정지** (동일 명의/디바이스 재가입 차단)
>
> 사용자에게는 외부 송금에 대한 환불·보상 의무 없음.

페널티 강도: 다이아 주선자 1년 활동 가치(약 240만원)를 1회 외부 송금(약 5만원)으로 잃는 구조 → 합리적 주선자 외부 유도 회피.

### B. UX nudge (자연스러운 앱 내 유도)

| 시점 | 노출 |
|---|---|
| 매칭 성공 화면 | "감사 팁 보내기" 버튼 prominent + 추천 금액(50/100/200 물감) |
| 매칭 D+3일 푸시 | "주선해주신 OO님께 감사 표시는 보내셨나요?" |
| 채팅 시작 화면 | "팁은 앱 내에서 안전하게 — 외부 송금은 사기 위험" 고지 |

→ 자연스러운 송금 동선 마련해서 외부 유도 동기 ↓.

### C. 채팅 키워드 탐지 (채팅 구현 시)

채팅 메시지에 다음 패턴 발견 시 메시지 자동 차단 + 관리자 알림 + 양쪽 토스트 경고:

```kotlin
val ACCOUNT_PATTERN = Regex("""\d{2,4}-\d{2,5}-\d{4,8}""")  // 계좌번호
val BANK_PATTERN = Regex("""(국민|신한|우리|하나|카카오|토스|농협|기업|새마을|우체국).*\d{6,}""")
val KEYWORDS = listOf("송금", "이체", "계좌", "직접", "현금", "토스로", "카카오페이로")
```

매칭당 2회 적발 시 주선자에게 1차 경고. 3회 적발 시 §A 페널티 적용.

베타 단계 채팅 mock 이라 본 ADR 시점엔 미구현 — Phase 2 채팅 구현 시 wire-up.

### D. 인센티브 정렬 (메인 deterrent)

앱 내 송금에만 다음 혜택 부여 — 외부 송금 시 주선자 본인 손해:

| 혜택 | 앱 내 | 외부 |
|---|---|---|
| 마일스톤 카운트 | ✅ +1 | ❌ |
| 등급 산정 통계 (성사·승률) | ✅ | ❌ |
| 출금 가능 잔액 적립 | ✅ | ❌ |
| 다음 매칭 풀 우선 노출 | ✅ | ❌ |
| 사용자 측 거래 이력·분쟁 보호 | ✅ | ❌ |

→ 외부 송금 5만원 수령 = 일시 +50,000원 (5천원 절약), but 등급 산정 -1건 + 풀 노출 ↓ + 출금 잔액 0
→ 합리적 주선자는 4.5만원(앱 내) + 등급/풀 보호 선택.

### E. 익명 신고 + 보상

`docs/DECISIONS/0033-trust-safety-domain.md` 의 `ReportService` 에 카테고리 추가:
- `EXTERNAL_PAYMENT_INDUCEMENT` — "외부 송금 유도" 신고 사유

신고 흐름:
1. 사용자가 채팅·통화에서 외부 송금 유도 받음
2. 앱 내 "신고하기" → 사유 선택 + 증거(스크린샷·통화 녹취) 첨부
3. T&S 패널 24h 내 검토
4. 확정 시 §A 페널티 + 신고 사용자에게 **50 물감 보상** (감사 인센티브)

## 결과

- 합리적 주선자 외부 유도 비율 5% 미만 (업계 일반 수준)
- 외부 유도 발각 시 페널티 강도가 1년 활동 가치 손실 → 시도 자체 억제
- 플랫폼 매출 보호 + 사용자 분쟁 보호 동시 달성
- 신고자 보상으로 자율 감시 메커니즘 형성

**Phase 3 (운영 데이터 누적 후)**:

## 메모

- "100% 차단 불가능"을 인정하고 인센티브 비대칭으로 합리적 행동 유도하는 게 핵심.
- §D 인센티브 정렬이 가장 비용 효율적 — 코드 변경 거의 없이 정책 1줄로 효과 큼.
- §A 페널티는 "발각 시 강하게" — 적발율 낮아도 deterrent 작용.
